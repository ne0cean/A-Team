- **Adversarial 체크**:
  - `serverByText.get(item.text)`: text=undefined인 item이 있으면 Map miss → 정상 처리(pass-through) ✓
  - 동적 arrayKeys: `_dismissed` 등 내부 필드 `_` prefix로 올바르게 제외 ✓
  - `incoming.days?.[day]` optional chaining으로 null 안전 ✓
  - **리스크**: item.text가 null인 경우 Map key가 null → serverByText.get(null) = undefined → done 복원 안 됨. 허용 가능 (text=null은 유효하지 않은 데이터)
