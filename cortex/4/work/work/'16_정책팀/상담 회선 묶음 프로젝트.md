상담 회선 묶음 프로젝트
		
		
	
	
		
			
매니저님, 
			
참고로~ 지난 번 정리해 주셨던 CRM센터 상담프로세스에 실제 어떤 번호에 
			
녹취가 남는지 C&amp;C의 피드백 내용 공유합니다.
			
그리고, 오전에 담당자와 통화하면서 녹취시스템이 개선에 어려움이 있다면 캠페인 상담사들이 DB에 있는 번호부터 통화를 시작해서 마지막 통화한 번호까지 모든 번호를 접촉이력 어디엔가 한꺼번에 저장할 수 있는 방법도 고민해 달라고는 했습니다.
			
추가 궁금하신 내용은 C&amp;C 임준구 과장과 협의하시죠~
			

			
**********************
			

			
캠페인 실행화면과 고객상담화면에서 특정이벤트 발생할때 레코딩으로 데이터전송을 하고 있습니다. 
			
물론 최종발생한 이벤트 기준으로 레코딩에는 저장됩니다.
			
캠페인 실행 화면
			
가. 걸기 클릭 시
			
나. 오퍼정보 변경 시
			
다. 고객인증성공 시
			
고객상담화면
			
A. 고객정보조회시
			
B. 상담이력 생성시
			
참고로 이동전화변경화면에서는 레코딩데이터 전송을 하지 않습니다.
			
업무 프로세스상에서 데이터 전송하는 시점을 Mapping 해보면
			
1. 캠페인DB로 발신시
			
ㅇ 사용하는 U.KEY화면 : 캠페인실행, 고객상담(유선/무선), 이동전화변경
			
ㅇ 캠페인실행화면에서 전화걸기 --&gt; 가 (C1고객)
			
→ 통화중 유/무선 결합을 위해 가족번호 조회는 고객상담(or 이동전화변경)화면에서 조회(조회 후 상담이력 생성) --&gt; A (C2고객) &gt; B (C2고객)
			
→ 최종 상담이력은 다시 캠페인실행화면으로 이동하여 생성함 --&gt; 생성없음
			
→ 하지만, 최종 상담이력 생성 후 통화종료 전 고객상담(or 이동전화변경)화면이 POP-UP되어 있으면,
			
고객상담(or이동전화변경)화면에서 조회되어 있는 번호가 최종 조회번호가 됨
			
: 결과적으로 C2고객에 대해서 레코딩 이력이 남습니다. 
			

			
2. 캠페인DB 외 번호로 발신시(가족번호로 발신시)
			
ㅇ 사용하는 U.KEY화면 : 고객상담(유선/무선), 이동전화변경
			
ㅇ 고객상담 화면에서 이동전화번호 조회 후 CTI툴바를 통해 발신 --&gt; 통화전 조회로 생성없음
			
→ 통화중 유/무선 결합을 위한 가족 이동전화번호 조회는 이동전화변경화면에서 조회(조회 후 상담이력생성) --&gt; B (C1고객)
			
→ 최종 상담이력은 최초 발신한 번호에 남겨야 하기 때문에 고객상담화면에서 이력생성 --&gt; B (C1고객)
			
→ 하지만, 고객상담화면에서 이력생성 후 통화종료 전 이동전화변경화면이 POP-UP되어 있으면, 
			
이동전화변경화면에서 조회되어 있는 번호가 최종 조회번호가 됨 --&gt; 이 부분 확인 필요합니다. 
			
고객상담에서 이력생성시 조회되어 있는 고객정보 기준으로 레코딩 데이터 생성됩니다. 
			
이동전화변경화면에서 C2를 조회한 상태라 하더라도 
			
다시 고객상담화면에서 C1으로 이력을 생성했다면 C1으로 레코딩 데이터 생성됩니다.
			
위 2개 케이스를 봤을때 2번케이스는 좀더 확인이 필요할것 같구요
			
1번 케이스에서 다시 캠페인실행화면에서 C1으로 캠페인이력생성시점에 레코딩 데이터를 추가로 생성하게 해준다면 최종 C1으로 레코딩데이터를 남기할 할 수 있을것 같습니다. 
		
		
			
1. 캠페인 DB 활용 시
			
				
- 복수의 화면이 띄워져 있을 경우 고객상담 화면에 우선하여 데이터가 남음
			
			
▶ 해당 상담 건을 위해 조회한 모든 번호를
			
    POP-UP 형태로 띄워서 선택적으로 묶을 수 있게 구현 가능한지
			

			
2. 캠페인 이외의 번호(가족번호로 발신 시)
			
				
- 이동전화변경/고객상담 화면 중 최종 이력 생성화면 기준으로 데이터 생성
			
			
▶ CTI툴바를 통해 발신한 내역에 대해 이력 생성 불가한지
			

			
즉, 1건의 상담을 시작과 끝맺는 시점은 상담사가 직접 지정(입력 해야겠지만)
			
해당 구간(시간)동안 진행한 조회 및 발신이력/상담녹취를 1개의 상담건으로 묶어줄 수 있으면 하는겁니다.
