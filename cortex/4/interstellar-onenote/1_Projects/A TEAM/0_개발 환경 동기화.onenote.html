    git pull￼

    +++++++


    ✅ 회사에서 안전하게 작업하는 5-명령어 루틴

    전제

  • 기본 브랜치: main
  • 회사에서는 절대 main에서 직접 작업 안 함
  • 항상 작업용 브랜치를 하나 만들어서 거기서만 작업

    • 1️⃣ 최신 상태로 맞추기 (시작 전)


    git pull￼


    2️⃣ 작업 브랜치 생성 + 이동 (핵심)


    git checkout -b work-company￼

    처음 한 번만 필요

    이후엔 git checkout work-company


    3️⃣ 작업 후 커밋 (회사에서)


    git add .￼git commit -m "회사: AI로 ○○ 수정"￼


    4️⃣ 브랜치 그대로 푸시


    git push -u origin work-company￼


    5️⃣ 맥북에서 최종 병합 (테스트 후)

    맥북에서만 실행:


    git checkout main￼git pull￼git merge work-company￼git push￼

    (병합 후 필요하면 브랜치 정리)


    git branch -d work-company￼git push origin --delete work-company￼


    🧠 이 루틴의 핵심 논리 (이해 포인트 30초)

  • main = 항상 “실행 가능 / 검증된 상태”
  • work-company = 실험실
  • 회사 AI가 망쳐도 → main은 안전
  • 충돌 나도 → 브랜치 단위로 처리해서 부담 ↓

    • 📌 정말 최소로 외울 것 (한 줄)

    회사에서는 무조건 checkout -b 작업브랜치부터


    충돌이 나도 안 무서운 이유

  • 충돌은 브랜치 머지할 때만 발생
  • 맥북에서 실행 테스트 가능한 상태에서만 해결
  • 꼬이면 언제든:

    • git merge --abort￼
