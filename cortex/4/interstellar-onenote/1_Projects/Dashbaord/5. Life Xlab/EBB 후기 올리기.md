  ② 참여 후기
			

			
  솔직히 말하면, 두 번 다 졌습니다. &#128517;
			

			
  #3 식목일 사무실 배치 — 9/15점(60%). 착수가 늦었고, 창가 배치 조건을
			
  오판했습니다. 속도와 정확도 둘 다 날린 회차였어요.
			

			
  #4 냥집사 대시보드 감사 — 더 뼈아팠습니다. 대시보드 오류 두 개(월매출
			
  7.75억→7.55억, ARPU 6,208→6,046원)까지 잡았는데, 정작 문제가 요구한 게 전체      
			
  ARPU가 아닌 S01 세그먼트 ARPU였습니다. 문제 지문을 제대로 읽지 않은 게
			
  원인이었어요. 정답은 6,455원이었는데 저는 다른 숫자를 냈습니다.
			

			
  느낀 점: AI 도구가 아무리 좋아도 문제를 잘못 읽으면 AI도 틀립니다. 속도보다      
			
  정확한 문제 해석이 먼저라는 걸 몸으로 배웠어요.
			

			
  좋았던 점: 매 회차가 실제 업무와 비슷한 데이터 구조라는 것. 회의실 예약, 경비    
			
  정산, 대시보드 검증 — 연습이 실전이 되는 느낌이었습니다.
			

			
  별로인 점: 대회 당일 파일이 여러 개 한꺼번에 오면 AI 컨텍스트가 터집니다.        
			
  (경험담입니다...) 파일 전달 방식에 대한 가이드가 있으면 좋겠어요.
			

			
  인상 깊었던 회차: #4. 틀렸지만 가장 많이 배웠습니다. 대시보드 수치를 원천        
			
  데이터로 역산해서 오류를 검출하는 패턴 — 이건 실무에서도 써먹을 수 있겠다        
			
  싶었어요.

  ③ 나의 AI 레시피
			

			
  저는 Claude Code (터미널) + ChatGPT Enterprise 듀얼 전략을 씁니다.
			

			
  워크플로우:
			
  1. Claude Code — 로컬에서 파이썬 스크립트를 직접 실행. 대회용으로 go.py 하나만   
			
  치면 파일 자동 탐색 → 패턴 분류 → 계산 파이프라인이 돌아가는 시스템을
			
  만들어뒀어요.
			
  2. ChatGPT Enterprise — Code Interpreter로 같은 파일을 병렬 분석. 두 AI가 다른   
			
  답을 내면 그게 곧 검증 신호입니다.
			
  3. 먼저 나오는 답 채택 — 속도 경쟁이니까요.
			

			
  핵심 팁 하나: AI한테 &quot;계산해&quot; 바로 시키지 말고, &quot;이 문제에서 계산 대상이 뭔지    
			
  먼저 정의해봐&quot; 라고 물어보세요. 저는 이걸 &#39;해석 게이트&#39;라고 부르는데, 이 단계를  
			
  건너뛰면 정답률이 반 토막 납니다. (#4에서 직접 경험)
			

			
  구독 중인 AI 툴:
			
  - Claude Code (Pro) — 코드 실행 + 분석, 메인 워크스테이션
			
  - ChatGPT Enterprise — Code Interpreter, 크로스체크 용도
			
  - 이유: 같은 데이터를 두 모델로 돌리면 한쪽이 놓친 걸 다른 쪽이 잡아냅니다. 단일 
			
  AI보다 확실히 정확도가 올라가요.
