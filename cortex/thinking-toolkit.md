# Thinking Toolkit

대화 중 발견된 개념, 접근법, 멘탈 모델을 축적한다.
새 항목 추가 시 출처(누가, 언제, 어떤 맥락)를 남긨다.
관련 상황에서 Claude가 선제적으로 꺼내준다.

---

## Concepts

### Unknown Unknowns
- **출처**: 2026-05-23, 에이전트 대화 중
- **정의**: 모르는 것조차 모르는 영역. Known knowns → Known unknowns → Unknown unknowns 순으로 위험도 증가.
- **적용**: 새 프로젝트 시작, 리스크 평가, 기획 단계에서 "내가 뭘 모르는지도 모르는 게 뭐가 있을까?" 질문

### MECE (Mutually Exclusive, Collectively Exhaustive)
- **출처**: 2026-05-23, Dean 직접 요구
- **정의**: 상호 배타적이고 전체를 빠짐없이 포괄하는 분류. 겹침 없이, 빠짐 없이.
- **적용**: 분류 작업, 옵션 비교, 문제 분해, 우선순위 정리 시 기본 프레임

## Ideation Frameworks (아이디어 발굴용)

> 제품/기능/사업 검토 시 3개 프레임을 순차 또는 병렬로 적용.
> `/office-hours`, `/brainstorm`, `/prd` 어디서든 호출 가능.

### Framework 1: Morphological Analysis (형태학적 분석)
- **출처**: Fritz Zwicky (제트 엔진 설계) + 2026-05-23 connectome 세션에서 적용
- **정의**: 독립된 축을 정의하고, 각 축의 값을 나열한 뒤, 모든 조합을 기계적으로 생성. 기존 아이디어가 어느 셀에 몰려있고 어느 셀이 비어있는지 시각화.
- **축 예시 (제품 설계용)**: 누가(주체) x 뭘(정보종류) x 어디서(공간범위) x 언제(시간) x 어떻게(상호작용) x 방향(정보흐름)
- **적용**: 기능 아이데이션, 빈 셀 = 미탐색 기회. 단위를 "주체x대상" 쌍으로 쪼개면 1세션 30분에 ~42셀 검토 가능.
- **주의**: 대부분 조합은 무의미. 빈 셀이 곧 기회는 아님. 흥미로운 빈 셀만 선별.

### Framework 2: Behavior Enumeration (행동 전수 조사)
- **출처**: 2026-05-23 connectome 세션
- **정의**: 특정 공간/맥락에서 사람이 하는 모든 행동을 나열하고, 각각이 디지털화되었는지 체크. 체크 안 된 행동 = 미탐색 기회.
- **적용**: "카페에서 사람이 하는 것 20가지" "출퇴근 중 하는 것 15가지" 등 맥락별 전수조사.
- **강점**: 형태학적 분석이 놓치는 미시적/구체적 행동을 잡아냄.

### Framework 3: Inversion (뒤집기)
- **출처**: Charlie Munger "Invert, always invert" + 2026-05-23 connectome 세션
- **정의**: 기존 아이디어의 모든 전제를 하나씩 뒤집음. "사람이 생성" → "사물이 생성", "능동적" → "수동적", "텍스트" → "음성" 등.
- **적용**: 기존 아이디어가 충분할 때 돌파구 찾기. 뒤집었을 때 말이 되는 조합이 혁신.
- **강점**: 형태학적 분석에 빠진 축 자체를 발견할 수 있음 (예: "이동/정지", "시각/청각" 축 발견).

### Framework 4: Human-Centric (인간 관점 프레임)
- **출처**: Dean 보유 프레임 (2026-05-23 언급)
- **정의**: 인간의 근본 축으로 빈 영역 탐색.
  - **의식주 + 디바이스**: 의(패션/자기표현) + 식(음식/카페) + 주(공간/장소) + 디바이스(폰/글래스/이어폰)
  - **오감 충족**: 시각(보기) + 청각(듣기) + 촉각(터치/햅틱) + 후각(향) + 미각(맛)
- **적용**: 제품이 어떤 감각/욕구를 충족하는지 매핑. 빠진 감각 = 미탐색 기회.
- **예시**: flair는 시각(프로필 보기) 중심. 청각(음성NPC, 라디오모드)은 공백 → 아이디어 도출.

### Framework 5: 12-Framework Startup Validation
- **출처**: Dean의 cortex 지식베이스 (원본: stdy.blog)
- **12가지**: (1)큰 시장 문제 선택 (2)현장 관찰 (3)세 질문 브레인스토밍 (4)비자명한 시장 (5)제약 완화 사고 (6)2주 공동창업 실험 (7)아이디어 숙성 (8)JTBD (9)비전+시점 정의 (10)4가지 검증기준(기능/감정/$1B+/혁신UX) (11)피드백 루프 가속 (12)열정 지속성
- **적용**: 아이디어 선별/검증 단계. `/office-hours` 스타트업 모드에서 활용.

### 프레임 사용 가이드

| 단계 | 프레임 | 목적 |
|------|--------|------|
| 발산 (빈 영역 찾기) | 1 형태학적 + 2 행동조사 + 3 뒤집기 + 4 인간관점 | 가능한 전체 공간에서 미탐색 영역 발굴 |

## 수집 프로토콜

### 1. 명시 트리거 (Dean 몫)
"이거 기억해", "이건 원칙이야", "이 접근법 좋다" → 즉시 저장

### 2. 자동 감지 (Claude 의무)
대화 중 다음 패턴 감지 시 **1줄로 질문**:
- 새로운 멘탈 모델, 프레임워크, 접근법 언급
- "~하게 해", "~식으로 접근해" 등 반복 가능한 원칙
- 다른 에이전트/외부 소스에서 온 유의미한 개념
- 실패에서 도출된 교훈

질문 형식: `toolkit 후보: [개념명] — 추가할까요?`
- 승인 → 이 파일에 append
- 거부 → 넘어감, 재질문 안 함

### 3. 세션 끝 수확 (/end 연동)
`/end` 실행 시 대화 전체에서 toolkit 후보 추출:
- 새 개념/접근법/교훈 목록 제시
- Dean이 승인/거부 선택
- 승인된 것만 이 파일에 append

## 활용 프로토콜

- **선제 제안**: 맥락에 맞는 항목이 있으면 1줄로 언급. ("이 상황에 [X] 접근법이 유용할 수 있습니다")
- **자가 점검**: 분류/비교/분석 작업 시 MECE 등 관련 원칙 자동 적용. 위반 시 알림.
- **중복 방지**: 기존 항목과 유사하면 병합.
- **프로젝트 횡단**: 모든 프로젝트 세션에서 이 파일 참조 + 기여.
