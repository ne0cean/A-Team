# Thinking Toolkit

대화 중 발견된 개념, 접근법, 멘탈 모델을 축적한다.
새 항목 추가 시 출처(누가, 언제, 어떤 맥락)를 남긨다.
관련 상황에서 Claude가 선제적으로 꺼내준다.

---

## Concepts

### Unknown Unknowns
- **출처**: 2026-05-23, 에이전트 대화 중
- **정의**: 모르는 것조차 모르는 영역. Known knowns → Known unknowns → Unknown unknowns 순으로 위험도 증가.
- **적용**: 새 프로젝트 시작, 리스크 평가, 기획 단계에서 "내가 뭘 모르는지도 모르는 게 뭐가 있을까?" 질문

### MECE (Mutually Exclusive, Collectively Exhaustive)
- **출처**: 2026-05-23, Dean 직접 요구
- **정의**: 상호 배타적이고 전체를 빠짐없이 포괄하는 분류. 겹침 없이, 빠짐 없이.
- **적용**: 분류 작업, 옵션 비교, 문제 분해, 우선순위 정리 시 기본 프레임

## Ideation Frameworks — 8 Frames (MECE)

> 아이디어 발굴의 전체 공간을 4방향 x 8프레임으로 MECE 커버.
> `/office-hours`, `/brainstorm`, `/prd` 어디서든 호출 가능.
> 출처: 2026-05-23 connectome 세션에서 정립 + MECE 검증.

```
아이디어는 어디서 오는가? (4방향 MECE)

1. 내부 확장 (가진 것 → 새 것)
   ├── F1. 형태학적 분석 — 축 조합의 빈 셀
   └── F2. 뒤집기 — 전제 반전

2. 외부 수입 (세상 → 우리)
   ├── F3. 행동 전수조사 — 현장 미시 행동 관찰
   ├── F4. 유추 — 타 산업/도메인 이식
   ├── F5. 기술 강제 — 신기술이 여는 문
   └── F6. 경쟁 공백 — 아무도 안 하는 것

3. 인간 본성 (욕구 → 기능)
   └── F7. 인간관점 — 의식주+디바이스, 오감

4. 제약 변형 (룰 변경 → 가능성)
   └── F8. 제약 놀이 — 제약 제거/추가 시뮬레이션
```

### F1: Morphological Analysis (형태학적 분석)
- **출처**: Fritz Zwicky (제트 엔진 설계)
- **방법**: 독립된 축을 정의 → 각 축의 값 나열 → 모든 조합 기계적 생성 → 빈 셀 = 미탐색 기회
- **축 예시 (제품용)**: 누가(주체) x 뭘(정보종류) x 어디서(공간범위) x 언제(시간) x 어떻게(상호작용) x 방향(정보흐름)
- **실행 팁**: "주체x대상" 쌍으로 쪼개면 1세션 30분에 ~42셀 검토 가능
- **주의**: 빈 셀이 곧 기회는 아님. 흥미로운 빈 셀만 선별

### F2: Inversion (뒤집기)
- **출처**: Charlie Munger "Invert, always invert"
- **방법**: 기존 아이디어의 전제를 하나씩 뒤집음. "사람→사물", "능동→수동", "텍스트→음성"
- **강점**: F1에 빠진 축 자체를 발견할 수 있음 (예: "이동/정지", "시각/청각" 축)

### F3: Behavior Enumeration (행동 전수 조사)
- **출처**: 2026-05-23 connectome 세션
- **방법**: 특정 맥락에서 사람이 하는 모든 행동을 나열 → 디지털화 안 된 행동 = 기회
- **실행 팁**: 맥락별로 쪼갬 (카페/출퇴근/쇼핑/여행/대학 등)
- **강점**: F1이 놓치는 미시적 행동을 잡아냄

### F4: Analogy (유추) — NEW
- **출처**: 2026-05-23 MECE 검증에서 공백 발견
- **방법**: "다른 산업의 X를 여기 적용하면?" 타 도메인 성공 패턴을 이식
- **예시**: "Uber의 매칭을 카페에? Duolingo의 스트릭을 장소에? Spotify Wrapped를 동선에?"
- **강점**: 검증된 패턴을 가져오므로 실패 확률 낮음

### F5: Tech Forcing (기술 강제) — NEW
- **출처**: 2026-05-23 MECE 검증에서 공백 발견
- **방법**: "새 기술 X가 가능하게 하는 것은?" 기술 변화에서 기회 도출
- **예시**: "LLM → 사물NPC 가능", "UWB → 1m 정밀 위치", "공간오디오 → 방향별 음성"
- **강점**: 타이밍 판단에 직결. "왜 지금?"에 대한 답

### F6: Competitive Whitespace (경쟁 공백) — NEW
- **출처**: 2026-05-23 MECE 검증에서 공백 발견
- **방법**: 경쟁자 전체를 나열하고, 모든 경쟁자가 안 하는 것을 찾음
- **예시**: "Strava도 NRC도 안 하는 것 = 장소인지 음성 가이드"
- **강점**: F1과 달리 시장 현실에 기반. 바로 실행 가능한 기회

### F7: Human-Centric (인간 관점)
- **출처**: Dean 보유 프레임
- **방법**: 인간의 근본 축으로 빈 영역 탐색
  - **의식주 + 디바이스**: 의(패션) + 식(음식) + 주(공간) + 디바이스(폰/글래스/이어폰)
  - **오감**: 시각 + 청각 + 촉각 + 후각 + 미각
- **강점**: 다른 프레임이 기능 중심일 때, 인간 욕구에서 역방향으로 찾음

### F8: Constraint Play (제약 놀이) — NEW
- **출처**: 2026-05-23 MECE 검증 + Dean cortex "제약 완화 사고"
- **방법**: 현재 제약을 나열하고, 하나씩 제거/추가했을 때 가능해지는 것을 탐색
- **예시**: "앱스토어 수수료 0%면?", "배터리 무한이면?", "GPS 실내 1m면?", "모든 사람이 AR글래스 쓰면?"
- **강점**: 미래 대비 + 현재 제약 우회 아이디어 동시 도출

### 프레임 사용 가이드

| 단계 | 프레임 | 목적 |
|------|--------|------|
| **발산** (빈 영역 찾기) | F1 형태학적 + F2 뒤집기 + F3 행동조사 + F4 유추 + F5 기술강제 + F6 경쟁공백 + F7 인간관점 + F8 제약놀이 | 가능한 전체 공간에서 미탐색 영역 발굴 |
| **수렴** (검증/선별) | 12-Framework Validation + /brainstorm ATTACK (Mom Test + 9차원 + Reality Score) | 생존 가능한 아이디어만 선별 |
| **테스트** | /brainstorm 프레토타입 유형 | 문짝/피노키오/YouTube MVP 등 |

### 12-Framework Startup Validation (수렴용)
- **출처**: Dean의 cortex 지식베이스 (원본: stdy.blog)
- **12가지**: (1)큰 시장 문제 (2)현장 관찰 (3)세 질문 (4)비자명한 시장 (5)제약 완화 (6)2주 실험 (7)아이디어 숙성 (8)JTBD (9)비전+시점 (10)4검증(기능/감정/$1B+/혁신UX) (11)피드백 루프 (12)열정 지속성
- **적용**: 발산 후 수렴 단계. `/office-hours` 스타트업 모드 + `/brainstorm` ATTACK과 병용

## 수집 프로토콜

### 1. 명시 트리거 (Dean 몫)
"이거 기억해", "이건 원칙이야", "이 접근법 좋다" → 즉시 저장

### 2. 자동 감지 (Claude 의무)
대화 중 다음 패턴 감지 시 **1줄로 질문**:
- 새로운 멘탈 모델, 프레임워크, 접근법 언급
- "~하게 해", "~식으로 접근해" 등 반복 가능한 원칙
- 다른 에이전트/외부 소스에서 온 유의미한 개념
- 실패에서 도출된 교훈

질문 형식: `toolkit 후보: [개념명] — 추가할까요?`
- 승인 → 이 파일에 append
- 거부 → 넘어감, 재질문 안 함

### 3. 세션 끝 수확 (/end 연동)
`/end` 실행 시 대화 전체에서 toolkit 후보 추출:
- 새 개념/접근법/교훈 목록 제시
- Dean이 승인/거부 선택
- 승인된 것만 이 파일에 append

## 활용 프로토콜

- **선제 제안**: 맥락에 맞는 항목이 있으면 1줄로 언급. ("이 상황에 [X] 접근법이 유용할 수 있습니다")
- **자가 점검**: 분류/비교/분석 작업 시 MECE 등 관련 원칙 자동 적용. 위반 시 알림.
- **중복 방지**: 기존 항목과 유사하면 병합.
- **프로젝트 횡단**: 모든 프로젝트 세션에서 이 파일 참조 + 기여.
