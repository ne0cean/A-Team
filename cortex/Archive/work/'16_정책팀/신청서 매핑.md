신청서 매핑
		
		
	
	
		
			
반영하기로 한 시스템은 U.Key 및 스마트플래너이며, 개발 내용은 아래와 같습니다.
			
현재까지 협의된 내용으로 인지하고 있는데 맞는거죠?
			
일단 아래 내용으로 7월초(7월6일예정)으로 진행하고 있습니다.
			

			
[U.Key]
			
1. 개인정보활용동의화면에서 &quot;고객요청/신청서확인&quot;사유로 동의처리함.
			
2. 동의처리 시 이미지를 첨부할 수있도록 함.
			
				
- 지점 및 대리점에서 업무처리하는 경우에 필수임. 
				
- 접촉이력 아래 항목이 조회되며, 첨부한 신청서 이미지를 확인할 수 있음.
				
-. 변경코드 : 동의여부변경(N2)
				
-. 사유코드 : 고객요청(03)
				
-. 상세항목(코드 값 셋팅)
				
업무처리채널(ACHG_ITM1) : U.key 또는 Smart Planner
				
정보광고 수신 동의여부 전/후(B/ACHG_ITM2) : 동의/미동의
				
고객 혜택 제공 동의여부 전/후(B/ACHG_ITM3) : 동의/미동의
			
			
3. 동의처리가 관련한 시스템은 U.Key 및 스마트플래너에서 가능함.
			

			
[스마트플래너]
			
1. 고객정보활용동의 여부 조회
			
- U.Key 연동하여 동의 여부 조회 (신규 TP)
			
- 기존 사용서비스 조회 화면에 동의 여부 표시 =&gt; &quot;광보/혜택 정보 수신 동의 전환 대상 고객입니다.&quot; 
			
. 항목이 복수가 될 수 있으므로 상세 내용 표시 필요
			
- 미동의 시, 동의(신청서 작성) 버튼 생성
			
=&gt; 사용서비스조회 UI 변경 필요
			
2. 동의 프로세스 
			
- 고객 SMS 인증
			
- 고객정보 활용동의 신청서 작성 및 U.Scan 전송 
			
- 동의 여부 U.Key 연동하여 업무처리(실시간)
			
- 스마트플래너에서 동의처리 후 U.Key에서 업무처리한 것과 동일하게 접촉이력 내역 확인 가능하며, 접촉이력에서 이미지 확인 가능함.
			
3. 신청서
			
- 광고정보수신동의 신청서 =&gt; 동의항목별 동의함/철회 체크하도록 이미지 변경 예정 
			
=&gt; 신청서에 별도 메뉴 생성 여부 확인
			
4.주요신청서 
			
- 선택약관(특정항목) 미선택 시, 다음 버튼 누를 경우 안내 팝업
			
=&gt; 반영 여부 재확인 필요
		
		
			
신청서 인쇄 및 배포시점 확인
