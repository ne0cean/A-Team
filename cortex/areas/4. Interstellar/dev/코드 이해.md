- “이 코드 줄별로 설명해줘”라고 하면 AI가 친절하게 설명해줍니다. 이제는 Stack Overflow 대신 LLM을 튜터로 삼는 시대예요.

코드 이해 (선택 사항)

AI가 생성한 코드를 완전히 이해할 필요는 없지만, 기본적인 부분을 알면 더 효과적으로 수정하고 발전시킬 수 있습니다. AI에게 "이 코드가 어떻게 작동하는지 간단히 설명해 줘"라고 물어보세요!

폴더 생성

mkdir -p /Users/noir/Desktop/HSC\_macro

  

프로젝트 전체 이동

mv /Users/noir/Desktop/CURSOR /Users/noir/Desktop/HSC\_macro/

  

코드 (터미널에서) 실행

  

0/ 항상 실행 전에 현재 위치 확인

Pwd

ls

  

1/  파일 실행하려는 폴더로 이동

/Users/noir/Desktop/CURSOR/

  

2/ 파일이 있는 폴더로 이동

Cd Src

  

3/ 파일 실행

python3 src/sniper\_macro.py

  

pwd

뜻

Print Working Directory

👉 내가 지금 “어디에 서 있는지” 보여줘

  

ls

뜻

List

👉 지금 폴더 안에 뭐가 있는지 나열

  

cd 경로

뜻

Change Directory

👉 다른 폴더로 이동

  

find . -name 파일명

뜻

👉 현재 위치(.)부터 파일명 찾기

find . -name "sniper\_macro.py"

  

python3 파일.py

뜻

👉 파이썬으로 이 파일 실행

python3 src/sniper\_macro.py

  

python3 -m 패키지.모듈

뜻

👉 폴더 구조 기준으로 실행 (안정적)

python3 -m src.sniper\_macro

  

which 명령어

뜻

👉 이 명령어가 실제로 어디에 있나?

which adb

  

설치 / 관리

brew install 패키지

뜻

👉 맥용 패키지 설치

brew install android-platform-tools

  

⚠️ 중단 / 복구

Ctrl + C

뜻

👉 지금 실행 중인 명령 강제 중단

  

adb kill-server

adb start-server

뜻

👉 ADB 재시작

  

pwd → 내 위치

ls → 주변 보기

cd → 이동

find → 수색

python3 → 실행

which → 어디 있나

PATH → 명령어 길
