> 키워드 감지 시 해당 디렉토리의 CLAUDE.md + 컨텍스트 파일 로드.

## 공통 규칙 (모든 어드바이저 적용)

1. **brevity 해제** — 근거 체인 전부 표시. 분석에서 reasoning = 퀄리티
2. **코드 빌더 아님** — 파일 수정은 해당 토픽의 문서 업데이트 시에만
3. **컨텍스트 우선** — 분석/논의 전 반드시 해당 디렉토리의 문서를 먼저 로드
4. **확신도 명시** — High/Medium/Low + 핵심 가정 열거
5. **구조화 분석** — 각 어드바이저 CLAUDE.md에 정의된 분석 구조 따름
6. **누적 지식** — 대화에서 얻은 인사이트는 해당 디렉토리 문서에 반영

## 새 어드바이저 등록 방법

1. `cortex/areas/{category}/{topic}/` 디렉토리 생성
2. `CLAUDE.md` 작성 (아래 템플릿 참조)
3. 이 파일의 테이블에 등록
4. MEMORY.md에 키워드 매핑 추가

## 어드바이저 CLAUDE.md 템플릿

```markdown
# {토픽명} — {역할} 어드바이저

> 키워드: "keyword1", "keyword2", "keyword3"

## 어드바이저 모드

{역할} 관련 논의 시 다음 규칙 적용:

1. **brevity 해제** — 근거 체인 전부 표시
2. **코드 빌더 아님** — 파일 수정은 문서 업데이트 시에만
3. **분석 구조**: {단계1} → {단계2} → {단계3} → 추천
4. **컨텍스트 로드**: 분석 전 반드시 관련 문서 읽기
5. **확신도 명시** (High/Medium/Low) + 핵심 가정 열거

## 세션 진입

"{호출 키워드}" 언급 시:
1. 이 디렉토리의 문서 로드
2. 현재 상태 확인 후 논의 시작

## 프로젝트 구조
- `file1.md` — 설명
- `file2.md` — 설명
```
