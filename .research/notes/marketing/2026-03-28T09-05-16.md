## 단계 1 — 제품 차별점 정리

### 실제 기능 목록 (코드 기반)

**입장 방식**
- 계정/회원가입 없음. 닉네임 + 아바타 색상만 선택 → 즉시 입장
- 프로필은 localStorage에만 저장 (서버 무저장)
- 3가지 입장 경로: GPS 모드 / 1-click 데모 / 프라이빗 룸 코드

**2D 공간 엔진**
- 어두운 캔버스 위에 컬러 orb로 사용자 표현
- GPS 모드: 반경 30m 실제 위치 → 캔버스 좌표 변환
- 프라이빗 룸: 룸 코드 or QR 스캔으로 공유
- 드래그로 자유 이동, 더블탭 → 핑 웨이브(파동 이펙트)

**실시간 상호작용**
- Shout (말풍선): 60자 캔버스 전체 공개 메시지, 일시적 소멸
- Reaction Burst: 특정 유저에게 이모지 반응
- Ping Wave: 더블탭 파동 — "여기 있어요" 신호
- DM: 요청→수락 컨센트 플로우, 수락된 상대와만 채팅
- Mood Picker: 이모지 기분 → 아바타에 실시간 표시

**프라이버시 4단계**
- Tier 1 익명: 색상 orb만 표시 (정체 불명)
- Tier 2 닉네임 공개
- Tier 3 상태메시지까지 공개
- Tier 4 연결 허용 + DM 가능

**부가 기능**
- 공간 오디오 (WebRTC 음성, 근접 시 소리 크게)
- QR 코드 공유 (초대 링크)
- 근접도 시각화 (Proximity Aura, Connection Line)
- Push Notification 지원

### 핵심 차별점 요약

| 경쟁사 대비 | Vibe Here |
|---|---|
| Gather.town, Kumospace | 계정 필수 → VH는 **무계정** |
| Discord, Slack | 텍스트 중심 → VH는 **공간 중심** |
| GPS 소셜 앱들 | 앱 설치 필요 → VH는 **PWA 웹** |
| Zoom, Meet | 일정 잡아야 → VH는 **항상 켜진 공간** |

**핵심 3줄 차별점**
1. **0 friction 입장** — 링크/QR 하나로 로그인 없이 즉시 동일 공간
2. **물리 공간 + 디지털 공간 융합** — GPS가 2D 캔버스 위치를 결정
3. **프라이버시 자기결정** — 익명 orb부터 풀 연결까지 본인이 선택

## 단계 2 — 포지셔닝 분석

### 2026년 "virtual presence" 시장 현황
- Gather.town, Kumospace: 원격 팀 오피스 → 기업용, 계정 필수, 무거움
- Luma AI / Spatial.io: 3D 메타버스 → 과기술, 진입장벽 높음
- BeReal, Zenly(종료): 위치 기반 소셜 → 앱 설치 + 친구 그래프 필요
- Discord Stage / Twitter Spaces: 오디오 기반 → 공간 개념 없음

**Vibe Here의 위치**: "ephemeral spatial web" — 설치도 계정도 없이 물리 장소나 링크로 순간 공간을 공유하는 틈새

### 적합 타겟 세그먼트 (우선순위순)

1. **개발자 해커톤/밋업** — 현장 + 원격 혼합 참가자, QR로 바로 공간 공유
2. **소규모 원격 팀 (2-8명)** — 계정 없이 "그냥 열어두는 공간" (always-on vibe room)
3. **대학 스터디 그룹** — 비동기 + 동기 혼합, 계정 만들기 귀찮은 Z세대
4. **이벤트/컨퍼런스 네트워킹** — 현장 GPS + QR로 공간 소셜 생성
5. **소규모 온라인 커뮤니티** — 특정 링크 룸을 커뮤니티 거점으로 운영

### 대안 태그라인 3개

**Option A — 마찰 제거 포커스**
> "링크 하나면 끝. 계정도 앱도 없이, 같은 공간에."
> *"One link. No account. No app. Same space."*

**Option B — 물리+디지털 융합 포커스**
> "지금 이 장소가 곧 공간이 됩니다."
> *"Your location becomes a room."*

**Option C — ephemeral/순간성 포커스**
> "만남은 순간이어도 충분합니다."
> *"Some moments don't need to be saved."*

## 단계 3 — 랜딩페이지 카피 초안

### 헤드라인
- **한글**: 링크 하나로 같은 공간에 — 계정도 앱도 없이
- **English**: Step in. No account. No app. Just a link.

### 서브헤드라인
> 링크를 공유하면 모두가 같은 2D 공간에 나타납니다.
> 이모지로 반응하고, 공간을 돌아다니고, 가까워지면 대화하세요.
>
> *Share a link. Everyone appears in the same 2D space.
> React, roam, and talk when you're close enough.*

### 핵심 가치 제안 3가지 (bullet)
- **가입 없음** — 닉네임 하나면 즉시 입장. 링크 받은 사람 누구나.
- **공간으로 연결** — 아바타를 움직이며 자연스럽게 가까워지는 느낌.
- **내가 정하는 프라이버시** — 익명 orb에서 DM 허용까지, 내 선택.

### CTA 버튼 텍스트 옵션 3개
1. `지금 바로 시작하기 →` (즉각성 + 행동 유도)
2. `1초 입장 — 계정 불필요` (마찰 제거 강조)
3. `내 공간 만들기` (소유감 + 창조 강조)

- 접근: "Vibe Here — Drop-in 2D spaces. No login, no app, just a link."
- 톤: 시각적 gif/데모 영상 중심, "1-click demo" 강조
- 키: 런치 당일 커뮤니티 동원 필요

**채널 3: Twitter/X 개발자 커뮤니티 (#buildinpublic)**

- 톤: 가볍고 솔직한 빌드인퍼블릭 톤
- 훅: 화면 녹화 10초 gif (아바타가 모이는 장면)

**채널 4: Reddit (r/remotework, r/digitalnomad, r/webdev)**
- 타겟: 원격근무자, 소규모 팀, 개발자

- 타겟: 해커톤, 밋업, 스터디 참가자
- 접근: "이 행사 참가자 전용 GPS 공간" — 현장 포스터/스티커에 QR
- 톤: "지금 이 공간에 있는 사람들과 연결하세요"
- 특이점: GPS 모드와 프라이빗 룸 모두 활용 가능한 유일한 채널

## 핵심 발견

1. **가장 강력한 USP는 "0 friction"** — 경쟁사 대비 진입장벽이 압도적으로 낮음. 이것이 모든 카피의 핵심이어야 함.

2. **GPS 모드는 킬러 피처지만 설명이 필요** — "반경 30m 안의 사람이 나타난다"는 개념이 낯설어서 데모나 gif가 필수.

3. **타겟을 2개로 명확히 분리 권장**:
   - Track A: 해커톤/밋업 → GPS 모드 + QR 오프라인 배포
   - Track B: 원격 소규모팀 → 프라이빗 룸 링크 공유

4. **현재 랜딩 카피 ("지금 이 공간의 사람들을 만나세요")** 는 GPS 모드에 치우쳐 있어 프라이빗 룸 유즈케이스를 놓침. 두 모드를 모두 커버하는 헤드라인 필요.

5. **프라이버시 4단계 시스템**은 경쟁사 없는 독창적 기능 — "나만의 공개 범위 선택"으로 마케팅 포인트화 가능.

## 개선 제안 (우선순위순)

1. **[높음]** 랜딩 헤드라인 A/B 테스트 — GPS 모드 강조 vs 링크 공유 강조 두 버전 운영, 전환율 측정 후 수렴
2. **[높음]** 10초 데모 GIF 제작 — 아바타가 등장하고 반응하고 DM하는 흐름. HN/PH/X 배포용
3. **[중간]** Show HN 포스트 작성 — "No-login 2D presence" 기술 스토리텔링
4. **[중간]** 오프라인 이벤트 파일럿 — 해커톤 1개에서 GPS 모드 실험, 실제 사용 데이터 확보
5. **[낮음]** 영문 랜딩 페이지 — 글로벌 얼리어답터 도달 위해 한/영 병행
