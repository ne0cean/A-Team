## 핵심 발견

### 🔴 크리티컬 버그

**`useMemo` 미import (RoomPage.jsx:1)**
- 파일 상단 import: `import { useEffect, useState, useRef, useCallback } from 'react'`
- 실제 사용: line 74, 79에서 `useMemo(...)` 호출
- React hooks는 반드시 import 필요. 런타임 `ReferenceError: useMemo is not defined` 발생
- 단, 앱이 현재 동작 중이라면 dev 환경 HMR 캐싱이나 전역 누출일 가능성도 있음 — 반드시 확인 필요

### 🟠 고빈도 Re-render 문제

**1. `users` 배열 매 move 이벤트마다 전체 교체**
- `useStore.updateUserPosition(id, x, y)` → `users.map(u => u.id === id ? {...u, x, y} : u)` → 새 배열 참조 생성
- RoomPage의 `const users = useStore(s => s.users)`는 배열 참조 변경을 감지하여 리렌더
- 결과: 유저 1명이 움직이면 **모든 Avatar 컴포넌트가 리렌더** (Avatar는 `memo`로 감싸져 있지만 props가 변하지 않아도 `usersWithOffsets` useMemo가 재계산됨)
- 소켓 `user_moved` 이벤트는 실시간으로 매우 빈번하게 발생

**2. ProximityAura — 매 render마다 O(n) 계산**
- `ProximityAura.jsx`의 `useWindowSize` 훅: 리사이즈 시 state 변경 → 컴포넌트 재렌더
- `memo`로 감싸져 있지만, `me`와 `users` props가 매 이동마다 변경됨
- render 내부에서 모든 유저에 대해 `Math.hypot` 계산 수행

**3. useCanvasInteraction — nearestUserId/proximityScaleMap/clusterOffsets 중복 계산**
- `nearestUserId`와 `proximityScaleMap`: `[me?.x, me?.y, users]` 의존
- `clusterOffsets`: `[users, me?.x, me?.y]` 의존
- `users` 배열이 매 이동마다 새 참조 → 두 useMemo 모두 매 move마다 재실행
- O(n²) 복잡도: clusterOffsets에서 `all.filter` → 전체 배열 2중 순회

**4. ConnectionLine — `window.innerWidth/Height` 직접 참조**
- `ConnectionLine.jsx:13-14`: `const ww = window.innerWidth; const wh = window.innerHeight`
- `memo`로 감싸져 있지만, 리사이즈 이벤트와 연동되지 않아 렌더 타이밍에 따라 stale 값 참조 가능
- ProximityAura와 달리 resize 리스너 없음 → 화면 회전 시 잘못된 거리 계산

### 🟡 구조적 문제
