## 핵심 발견

### ❌ 1. Push Subscription SSRF — 높은 위험 (즉시 수정)

- **위치**: `server.js` push_subscribe 핸들러
- **취약점**: endpoint URL이 도메인 검증 없이 저장됨
- **공격 시나리오**:
  1. 공격자가 `push_subscribe`로 `endpoint: "http://192.168.1.1/admin"` 등록
  2. 다른 사용자가 DM 요청
  3. 서버가 내부 IP 주소로 WebPush POST 요청 전송 → 내부 네트워크 정보 유출
- **영향도**: 심각 (원격 코드 실행 가능성까지)

### ❌ 2. x-forwarded-for 헤더 스푸핑 — 높은 위험

- **위치**: IP 추출 로직 전체 (HTTP rate limit, MAX_CONNECTIONS_PER_IP 등)
- **현재 코드 패턴**:
  ```javascript
  const ip = req.headers['x-forwarded-for']?.split(',')[0].trim() || req.socket.remoteAddress;
  ```
- **취약점**: reverse proxy 없는 직배포 환경에서 헤더 신뢰
- **공격 시나리오**: X-Forwarded-For 헤더 위조로 IP 기반 rate limit 우회, MAX_CONNECTIONS_PER_IP(5) 우회
- **영향도**: IP 기반 방어 전부 무력화

### ⚠️ 3. DM 세션 공간 이탈 후 지속 — 중간 위험

- **위치**: `send_dm_message` 핸들러
- **취약점**: `activeDmSessions`가 공간 이탈 후에도 유지됨
- **공격 시나리오**:
  1. 사용자 A + B → DM 수락 (activeDmSessions 생성)
  2. A가 GPS 공간 leave
  3. B가 계속 DM 전송 → A에게 도달 가능
- **영향도**: 공간 경계 의미 무화, 설계 의도와 불일치

### ⚠️ 4. 입력 검증 — HTML Escape 부재

- **위치**: `sanitizeStr` 함수
- **현재**: 길이 제한만 (`trim().slice(0, maxLen)`)
- **취약점**: `<`, `>`, `&`, `"`, `'` 미처리
- **현재 위험도**: 낮음 (React 자동 escape 적용 중)
- **잠재 위험**: 비React 클라이언트 추가 시 XSS 취약

### ⚠️ 5. Emoji 화이트리스트 미적용 — 낮음~중간

- **위치**: `send_reaction` 핸들러
- **취약점**: 길이(10자) 제한만, 유효성 검증 없음
- **공격 시나리오**: ZWJ 시퀀스 조합으로 클라이언트 렌더링 오류, UI 교란

### ⚠️ 6. WebRTC SDP 크기 제한 부재 — 낮음~중간

- **위치**: `webrtc_offer`, `webrtc_answer` 핸들러
- **취약점**: `typeof string` 확인만, 크기 제한 없음 (maxHttpBufferSize 100KB가 상한)
- **영향도**: 대용량 SDP로 타겟 클라이언트 부담

### ℹ️ 7. 룸 설정 인가 부재 — 낮음

- **위치**: `set_room_settings` 핸들러
- **현재**: 룸 입장자 누구나 이름/설명 변경 가능 (owner 개념 없음)

### ℹ️ 8. CORS 환경변수 미설정 시 localhost 허용 — 낮음

- **현재**: `CLIENT_URL` 미설정 시 `http://localhost:5173` 허용
- **위험**: 프로덕션 배포 시 환경변수 누락 → 의도치 않은 origin 허용

## 개선 제안 (우선순위순)

### 1. **[높음/즉시]** Push Subscription SSRF 방어
- **수정 대상**: `server.js` — push_subscribe 핸들러
- **예상 난이도**: 낮음 (15~30분)
- **수정 방향**:
  ```javascript
  const ALLOWED_PUSH_HOSTS = /^https:\/\/(fcm\.googleapis\.com|.*\.push\.services\.mozilla\.com|.*\.notify\.windows\.com|.*\.push\.apple\.com)/i;

  socket.on('push_subscribe', safe((subscription) => {
    if (!subscription?.endpoint) return;
    if (!ALLOWED_PUSH_HOSTS.test(subscription.endpoint)) return;
    pushSubscriptions.set(socket.id, subscription);
  }));
  ```

### 2. **[높음/즉시]** x-forwarded-for 신뢰 정책 명시
- **수정 대상**: `server.js` — IP 추출 로직 전체
- **예상 난이도**: 낮음 (30~40분)
- **수정 방향**:
  ```javascript
  const TRUST_PROXY = process.env.TRUST_PROXY === 'true';

  function getClientIp(req) {
    if (TRUST_PROXY) {
      return req.headers?.['x-forwarded-for']?.split(',')[0].trim()
        || req.socket?.remoteAddress;
    }
    return req.socket?.remoteAddress;
  }
  ```
- **배포 필요**: nginx에서 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` + `.env`에 `TRUST_PROXY=true`

### 3. **[높음/즉시]** DM 세션 공간 재검증 강화
- **수정 대상**: `server.js` — send_dm_message 핸들러
- **예상 난이도**: 낮음 (20분)
- **수정 방향**: `isSameSpace` 체크를 DM 전송 시에도 재실행 + GPS 공간 이탈 시 activeDmSessions 정리

### 4. **[중간]** Emoji 화이트리스트 적용
- **수정 대상**: `server.js` — send_reaction 핸들러
- **예상 난이도**: 낮음 (15분)
- **수정 방향**:

  ```javascript
  const VALID_REACTIONS = ['👍','❤️','😂','😮','😢','🔥','👏','🎉','🤔','✨'];
  if (!VALID_REACTIONS.includes(emoji)) return;
  ```

### 5. **[중간]** WebRTC SDP 크기 제한
- **수정 대상**: `server.js` — webrtc_offer, webrtc_answer 핸들러
- **예상 난이도**: 낮음 (15분)
- **수정 방향**:
  ```javascript

  if (typeof offer.sdp !== 'string' || offer.sdp.length > MAX_SDP_SIZE) return;
  ```

### 6. **[중간]** 프로덕션 환경변수 필수 검증
- **수정 대상**: `server.js` 상단
- **예상 난이도**: 낮음 (20분)
- **수정 방향**:

  ```javascript
  if (process.env.NODE_ENV === 'production') {
    const required = ['CLIENT_URL', 'VAPID_PUBLIC_KEY', 'VAPID_PRIVATE_KEY'];
    const missing = required.filter(k => !process.env[k]);
    if (missing.length > 0) { console.error(`Missing: ${missing.join(', ')}`); process.exit(1); }
  ```

### 7. **[낮음]** HTML Escape 서버 레벨 추가 (심층 방어)

- **수정 방향**: `.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;')` 체인 추가
