## 단계 2 — 위협 모델 분석

### 🔴 높음

#### 1. GPS 위치 스푸핑 (Location Spoofing)
**파일**: server.js:302 (`gps_update`), server.js:242 (`join_gps`)
**설명**: 클라이언트가 임의의 lat/lng를 전송 가능. `validateLatLng()`는 범위(-90~90, -180~180)와 타입만 검증. 실제 물리적 위치를 증명하는 수단 없음.
**공격 시나리오**:
- 서울 강남 좌표를 위조해 해당 지역 privacyTier 3 유저들의 정밀 좌표를 수집
- 특정 지역을 순회하며 gpsUsers 전수 조사 가능 (gps_update 1초 rate limit만 존재)
- 1초 간격으로 좌표를 이동시키며 전체 GPS 공간 탐색 가능

#### 2. Push Subscription 객체 주입
**파일**: server.js:511 (`push_subscribe`)
**설명**: `subscription.endpoint` 존재 여부만 확인. 악의적 subscription 객체 전체가 `webpush.sendNotification()`에 그대로 전달됨.
**공격 시나리오**:
- 공격자가 자신이 제어하는 서버 URL을 endpoint로 설정
- 피해자가 DM 요청/메시지를 보내면, 알림 페이로드(닉네임 + 메시지 80자)가 공격자 서버로 전달됨
- push subscription 유효성 검증(`endpoint` URL 패턴, 프로토콜, 도메인 확인) 없음

### 🟡 중간

#### 3. X-Forwarded-For 헤더 스푸핑
**파일**: server.js:66, 75, 87, 224 (IP 추출 로직)
**설명**: `req.headers['x-forwarded-for']?.split(',')[0].trim()`을 신뢰. 역방향 프록시(nginx 등) 없이 직접 노출된 경우 클라이언트가 헤더를 조작 가능.
**공격 시나리오**:
- IP 기반 HTTP rate limit 우회
- IP당 연결 제한(5개) 우회 → 단일 IP에서 무제한 연결

#### 4. Room Enumeration
**파일**: server.js:74 (`/api/rooms/:code/count`)
**설명**: 룸 코드 길이는 6자리 hex (16^6 ≈ 1677만 조합). `/api/rooms/:code/count` API는 rate limit 3초이나, 분산 IP에서 브루트포스 가능.
**공격 시나리오**:
- 활성 룸 목록 열거 후 인원이 있는 룸에 join_space로 진입
- 개인 룸이라도 코드만 알면 입장 가능 (별도 비밀번호/초대 없음)

#### 5. 이모지 반응 내용 미검증
**파일**: server.js:695 (`send_reaction`)
**설명**: emoji 필드를 `sanitizeStr(emoji, MAX_EMOJI)`로 길이만 제한. `VALID_MOODS`처럼 allowlist 없음.
**공격 시나리오**:
- 임의의 유니코드 문자열 10자 전송 가능
- 클라이언트 렌더링 방식에 따라 ZWJ 시퀀스나 특수 문자로 UI 오동작 유발 가능

#### 6. WebRTC SDP 내용 무검증
**파일**: server.js:559, 566, 573
**설명**: `typeof offer.sdp !== 'string'`만 검증. SDP 내용 자체는 무검증으로 릴레이.
**공격 시나리오**:
- 대용량 SDP 문자열 전송 시 relay 버퍼에 부하 (단, maxHttpBufferSize 100KB 제한이 일부 완화)
- SDP에 포함된 내부 LAN IP 노출을 통한 네트워크 구조 파악 (서버가 아닌 클라이언트 간 교환이므로 주로 클라이언트 위험)

### 🟢 낮음

**설명**: disconnect 이벤트에서 clearRateLimit 호출. GPS 모드와 Private 모드 모두 정상 정리됨. 그러나 disconnect 없이 연결이 중단된 경우(네트워크 단절 등) `pingTimeout: 5000` 내에 정리됨.

## 단계 3 — 설정 검토

### CORS
- **현재**: `process.env.CLIENT_URL || 'http://localhost:5173'` 단일 origin
- **프로덕션 적합성**: 환경변수 미설정 시 localhost가 origin으로 사용 → 모든 프로덕션 요청 차단
- **권고**: 배포 체크리스트에 `CLIENT_URL` 환경변수 필수 설정 명시

### 메모리에 저장되는 민감 데이터
| 데이터 | 저장 위치 | 보존 기간 | 위험도 |
|--------|----------|----------|--------|
| 실제 GPS 좌표 (lat/lng) | `gpsUsers` | 접속 중 | 높음 — privacyTier 3이면 타 사용자에게 노출 |
| 닉네임 | `gpsUsers`, `privateRooms` | 접속 중 | 중간 |
| Push subscription (endpoint + keys) | `pushSubscriptions` | disconnect 후 30분 | 중간 — endpoint URL 포함 |
| IP 주소 | `ipConnectionCount`, `socket.data._ip` | 접속 중 | 낮음 — 연결 관리용만 사용 |
| DM 메시지 내용 | 저장 안 함, 릴레이만 | 없음 | 양호 |

### 인증/인가 부재
- 모든 연결이 익명. socket.id가 유일한 식별자
- socket.id는 Socket.io가 자동 생성 (추측 불가)
- **위험**: 재연결 시 새 socket.id 발급 → DM 세션/위치 정보 소실 (설계적 한계)
- **위험**: 재연결한 악의적 사용자가 이전 피해자의 socket.id를 알아도 새 연결로는 impersonation 불가 (socket.id는 서버 측 생성) → 양호

## 개선 제안 (우선순위순)

### 즉시 수정 필요

1. **[높음] Push Subscription 유효성 검증 추가**
   - **파일**: `server.js:511`
   - **수정 방향**: endpoint URL이 `https://` 프로토콜이고 알려진 푸시 서비스 도메인(fcm.googleapis.com, updates.push.services.mozilla.com, notify.windows.com 등)에 속하는지 확인. 또는 최소한 URL 파싱 후 프로토콜/호스트 검증.
   - **예시 검증 로직**:
     ```js
     const ALLOWED_PUSH_DOMAINS = ['fcm.googleapis.com', 'updates.push.services.mozilla.com', 'notify.windows.com'];
     const url = new URL(subscription.endpoint);
     if (url.protocol !== 'https:') return;
     if (!ALLOWED_PUSH_DOMAINS.some(d => url.hostname.endsWith(d))) return;
     ```
   - **난이도**: 낮음

2. **[높음] X-Forwarded-For 신뢰 설정 강화**
   - **파일**: `server.js:66, 75, 87, 224`

   - **난이도**: 낮음

4. **[중간] GPS Rate Limiting 강화 (위치 스푸핑 억제)**

   - **수정 방향**: 이전 위치와 현재 위치의 속도 계산. 예: 1초에 100km 이상 이동은 불가능. 위반 시 업데이트 무시.
   - **예시 로직**:
     ```js
     const prevLat = user.lat, prevLng = user.lng;
     const dist = haversineDistance(prevLat, prevLng, lat, lng);
     const elapsed = Date.now() - (user._lastUpdate || 0);
     const speedMps = dist / (elapsed / 1000);
     if (speedMps > 100) return; // 360km/h 이상은 위조로 간주
     user._lastUpdate = Date.now();
     ```
   - **난이도**: 낮음

5. **[중간] Room Code 길이/엔트로피 증가**
