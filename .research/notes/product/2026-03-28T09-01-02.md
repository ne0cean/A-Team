## 단계 3 — 신규 기능 아이디어 (3가지)

### 아이디어 1: Vibe Zones 2.0 (기분 기반 색상 아우라 강화)
- **핵심 기능**:
  1. 같은 mood 사용자 주변에 반투명 컬러 그라데이션 존 렌더링 (캔버스 레이어)
  2. 우측 패널에 mood 카운터 미니 대시보드: 🔥3명 😌2명 💫1명
  3. 내 mood와 같은 유저에게 subtle 하이라이트 효과 (pulse ring 색상 변경)
- **차별화**: 군중 감정 시각화 — 텍스트 없이 공간의 "바이브"를 직관적으로 전달. lightweight spatial presence 컨셉과 완벽히 부합
- **구현 방법**:
  - Avatar.jsx의 기존 `GLOW` 색상 팔레트에 mood 오버레이 레이어 추가
  - useStore에 `moodGroups` computed selector 추가 (별도 socket 이벤트 불필요)
  - SpacePage/RoomPage에 SVG/CSS 기반 mood 존 배경 레이어 추가
- **난이도**: **소-중** (2-3h) — 기존 mood/GLOW 인프라 재활용

### 아이디어 2: Presence Timeline (30초 이동 자국)
- **핵심 기능**:
  - 지난 30초 내 사용자 위치 이력을 페이드아웃 점선으로 캔버스에 표시
  - 내 아바타는 선명하게, 타인은 30% opacity로 렌더링
  - 모바일 드래그/GPS 이동 양쪽 지원
- **차별화**: 공간형 presence를 "이 사람이 방금 어디서 왔나"로 직관화. 움직임이 드문 공간에서도 활동 흔적 제공
- **구현 방법**:
  - useStore의 `updateUserPosition`/`updateUserGps` 에서 히스토리 배열(최대 15개 좌표) 유지
  - SpacePage/RoomPage 캔버스에 `<svg>` polyline + opacity gradient 추가
  - requestAnimationFrame 기반 페이드아웃 (0.5초마다 30→20→10→0% opacity)
- **난이도**: **중** (3-4h) — Canvas 히스토리 관리 + 성능 최적화 필요

### 아이디어 3: Instant Reactions Wall (공간 공유 감정 보드)
- **핵심 기능**:
  - 아바타 위쪽 공중에 리액션 이모지들이 실시간으로 떠오르는 "감정 대기"
  - 공간 진입 시 지난 5분 리액션 히스토리를 반투명 흔적으로 표시 (신규 유저 컨텍스트 제공)
  - 탭/클릭으로 자신의 리액션 보내기 + 타인 리액션에 "공명" (같은 이모지로 응답)
- **차별화**: 단방향 리액션을 "공명 루프"로 전환. 말 없이도 감정적 연결감 형성. Discord Stage처럼 큰 그룹에서 효과적
- **구현 방법**:
  - 서버에 `reactionHistory[]` (5분 TTL) 유지
  - 신규 소켓 이벤트: `get_reaction_history` (공간 입장 시), `resonate_reaction` (공명 클릭 시)
  - ReactionBurst.jsx에 "공명 카운터" 뱃지 추가 (기존 컴포넌트 확장)
- **난이도**: **중** (4-5h) — 서버 히스토리 로직 + UI 공명 인터랙션 신규

## 기술 노트

### useStore 현재 구조 분석
- `me`, `users`, `roomId`, `socket`, `privacyTier`, `canvasMode` — 기본 상태
- DM 상태 별도 관리: `dmRequest`, `activeDm`
- **누락**: `moodHistory`, `positionHistory`, `encounterLog` — 신규 기능에 필요
- `users` 배열에 `mood` 필드는 `user_mood` 소켓 이벤트로 이미 업데이트됨 → Vibe Zones 바로 구현 가능

### Avatar.jsx Mood 인프라
- `MOUTH` 맵: 6가지 mood별 SVG mouth path (😌🔥💫🎉😢😤)
- `GLOW` 맵: 6가지 avatar 색상별 drop-shadow
- 현재 mood는 아바타 좌상단 뱃지로만 표시 — 아우라/환경 반영 없음
- **Vibe Zones 진입점**: `isNearest` pulse ring 로직을 mood 매칭 로직으로 확장 가능

### SummaryCard 구현 현황
- html2canvas로 이미지 저장 + navigator.share로 공유 — 완성도 높음
- `encountered` 배열에 `joinedAt` timestamp 있으나 현재 duration 계산에만 사용
- **확장 가능**: mood 분포, 리액션 통계, Voice Chat 시간을 SummaryCard에 추가 가능
