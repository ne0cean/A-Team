## 단계 3 — 신규 기능 아이디어 (lightweight spatial presence 컨셉)

### 아이디어 A: "Mood Trail" — 이동 궤적 시각화
**핵심 기능**: 아바타가 이동하면 0.5-2초간 희미한 컬러 발자국 흔적이 남음 (CSS opacity fade-out). 방금 어디서 왔는지, 얼마나 활발하게 움직이는지 한눈에 보임.
**차별화 이유**: 공간에 "역사"를 부여 — 정적인 오브들이 아니라 살아있는 존재감. Gather.Town/Kumospace에는 없는 가벼운 differentiator.
**구현 방법**:
- `SpacePage.jsx`, `RoomPage.jsx` — `trailsMap: Map<userId, [{x,y,ts}]>` 상태 추가
- CSS: `position:absolute; border-radius:50%; opacity: 0.4→0; transition: opacity 1s;`
- `user_moved` 수신 시 trail 포인트 push, 2초 후 자동 제거
- 서버 변경 없음 (클라이언트 전용)
**구현 난이도**: 소 (3-5시간)

### 아이디어 B: "Vibe Check" — 실시간 집단 감정 온도계
**핵심 기능**: 룸/공간 전체의 현재 Mood 분포를 우상단 히트맵 또는 파이 차트로 표시. "지금 이 공간의 바이브: 🔥 60% 신남 / 😴 20% 힐링 / 🎯 20% 집중중". 개별 Mood를 집계해 공간 전체의 분위기를 수치화.
**차별화 이유**: 소셜 플랫폼에서 "지금 이 공간이 어떤 에너지인가"를 직관적으로 전달. 신규 입장자의 진입 불안을 낮추는 FOMO 강화 수단.
**구현 방법**:
- `server.js` — mood_distribution 계산 (기존 user.mood 필드 활용)
- 새 소켓 이벤트 `vibe_check` — 인원 변동 / mood 변경 시 브로드캐스트
- 신규 컴포넌트 `VibeThermometer.jsx` — Framer Motion 애니메이션 바
- `SpacePage.jsx`, `RoomPage.jsx` — 헤더 또는 좌측 사이드에 Thermometer 마운트
**구현 난이도**: 중 (6-10시간)

### 아이디어 C: "Temporal Rooms" — 자동 소멸 방
**핵심 기능**: 방 생성 시 수명 선택 (1시간 / 24시간 / 7일). 만료 시 자동 삭제, 남은 시간 타이머 표시. "지금만 있는 방" → 희소성 심리 자극.
**차별화 이유**: 영속 DB 없이도 "의미 있는 방"을 만들 수 있게 함. Room Settings 영속성 구현 전 중간 단계로도 유용. Discord 임시 채널, Snapchat 스토리 컨셉의 공간 버전.
**구현 방법**:
- `server.js` — `privateRooms[code]._expiry = Date.now() + duration` 추가
- `setInterval` 5분마다 만료 방 청소 (기존 패턴과 동일)
- 새 소켓 이벤트 `room_expiry_warning` — 5분 전 경고
- `RoomPage.jsx` — 헤더에 카운트다운 타이머
- `ProfileForm.jsx` / `ModeSelect.jsx` — 방 생성 시 duration 선택 UI
**구현 난이도**: 소-중 (4-7시간)
