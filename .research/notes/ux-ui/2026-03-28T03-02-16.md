## 단계 1 — 2D 인터랙션 구현 방식

### 이동 로직
- 위치 좌표계: `vw/vh %` 기반 (0–100 범위, 실제 clamp: 5–95)
- 드래그 스로틀: `80ms` 간격으로 소켓 `emit('move')` — 네트워크 부하 적정
- 드래그 위치 계산: `info.point` (page 좌표) → `rect` 기준 상대값 → zoom 보정 → % 변환
- 소켓 emit 전략: 드래그 중 throttle emit + dragEnd 시 final emit

### 클러스터 팬아웃 (cluster fan-out)
- `THRESHOLD = 5` (% 단위) 이내 겹치는 아바타들을 원형으로 분산
- 반경 공식: `radius = 3.5 + cluster.length * 0.5` — 사람 많을수록 벌어짐
- 타원형 보정: `y * 0.7` (수직 방향 약간 압축)
- 분산 결과가 `clusterOffsets` → `meWithOffset`, `usersWithOffsets`로 병합

### 근접 감지
- `MAX_DIST = min(vw, vh) * 0.15` — 화면 크기 15% 반경
- `nearestUserId`: 가장 가까운 유저 1명만 추적
- `proximityScaleMap`: 근접할수록 scale 최대 +15% 증가 (`1 + 0.15 * (1 - dist/maxDist)`)

### 더블탭 감지
- `lastTapTime.current`: 350ms 이내 두 번 탭 = `ping_wave` emit
- 단순 ref 비교 방식 — 경량이지만 Touch 이벤트와 Mouse click 혼용 시 오작동 가능성 있음

## 단계 2 — 컴포넌트 애니메이션 분석

### Framer Motion spring 값 (Avatar.jsx)

| 상수 | stiffness | damping | mass | 용도 |
|------|-----------|---------|------|------|
| `SPRING_ENTER` | 350 | 22 | 1.1 | 아바타 등장 + opacity/rotate/scale |
| `SPRING_MOVE` | 180 | 28 | 0.8 | 다른 유저 이동 추적 |
| `BOUNCE_HOVER` | 400 | 12 | — | 호버 bounce |

### 평가

**SPRING_ENTER (350, 22, 1.1)**
- mass=1.1이 약간의 무게감 부여. damping=22는 오버슈트 약간 있음.
- "가볍고 빠른" 느낌에는 적합. mass를 0.8로 낮추면 더 경쾌해짐.
- 현재: 괜찮음 ✅

**SPRING_MOVE (180, 28, 0.8)**
- damping=28은 오버댐프에 가까움 → 다른 유저 이동이 "스르르" 따라옴
- stiffness=180은 부드럽지만 반응성은 낮음
- 제안: stiffness를 220으로 올리면 더 즉각적인 추적감, damping 24로 낮추면 약간의 탄성
- **개선 여지 있음** ⚠️

**BOUNCE_HOVER (400, 12)**
- damping=12 → 매우 탄성적. 호버 시 큰 바운스 발생
- Vibe Here의 "가볍고 귀여운" 컨셉에는 맞지만, 실용성 있는 2D 공간 앱으로 방향성이 간다면 damping을 18~20으로 올리는 게 안정적
- 현재: 컨셉에 맞음 ✅ (취향 선택)

**Idle bob 애니메이션**
- `y: [0, -5, 0]`, 2.4초, `bobDelay`로 아바타별 stagger
- 부드럽고 캐릭터감 있음 ✅
- stagger 방식 (charCode % 10 * 0.28초)은 기발하고 가벼움 ✅

**EphemeralShout**
- 3.5초, keyframe `[0, 0.08, 0.75, 1]` — 빠른 나타남, 길게 유지, 부드럽게 사라짐
- `times` 비율 적절. 메시지 읽기 충분한 시간 확보 ✅

**트랜지션 타이밍 제안 요약:**
1. `SPRING_MOVE`: `stiffness: 220, damping: 24` → 타 유저 이동 추적 더 responsive
2. `SPRING_ENTER` mass: `1.1 → 0.85` → 등장 더 경쾌
3. `BOUNCE_HOVER` damping: `12 → 16` → 호버 바운스 약간 안정화 (취향에 따라)

## 단계 3 — 접근성 & 모바일 감사

### ✅ 잘 구현된 부분

**aria-label 적용:**
- 헤더 버튼 전체 (Copy, Privacy, Mood, Shout, Mic, Leave)에 `aria-label` + `title` 있음
- Non-me Avatar: `role="button"`, `tabIndex={0}`, `aria-label="${nickname}과 상호작용"`, `onKeyDown(Enter)` 처리 완비
- UserMenu 리액션 버튼: `aria-label={label} 리액션 보내기` 있음
- `focus-visible:ring-2 focus-visible:ring-vibe-400` 키보드 포커스 표시 일관됨
- Escape 키로 모든 모달/메뉴 닫기

**모바일 최적화:**
- `touch-none` (내 아바타 드래그), `touch-manipulation` (타 아바타)
- `dragMomentum={false}` — 모바일 드래그 후 관성 없음 (의도적 위치 제어에 적합)
- status 버블 `[@media(pointer:coarse)]:opacity-100` — 터치 기기에서 항상 표시

### ❌ 누락/개선 필요

**키보드 접근성 누락:**
1. `UserMenu` "DM 보내기" 버튼: `aria-label` 없음 (텍스트는 있지만 스크린리더 컨텍스트 부족)
2. `UserMenu` "리액션" 버튼: `aria-label` 없음
3. **내 아바타(isMe)**: `tabIndex` 없음 → 키보드로 포커스/클릭 불가
   - 더블탭(ping) + 단탭(프로필 편집) 인터랙션이 키보드로 불가능

**이동 접근성:**
- 드래그 이동만 지원 → 키보드/switch access 사용자는 위치 이동 불가
- 화살표 키 이동 (예: 5% 씩 이동) 없음

**모바일 이동 UX:**
- 핀치줌: `enableZoom=false`가 RoomPage 기본값 → RoomPage에서는 줌 없음
  (SpacePage는 `enableZoom=true`로 차별화)
- 드래그 이동: 모바일에서 아바타가 작아서 정밀 드래그 어려움 (52px 크기)
- 헤더 버튼 터치 타겟: `px-4 py-3` ≈ 44px 높이 → 모바일 최소 권장치(44px) 경계선

**색상 대비:**
- nickname pill: `text-[10px]` + `text-white/slate-950` 배경 — 아주 작은 텍스트
- status badge: `text-[9px]` — 9px는 접근성 기준(12px 최소) 미달
- Aurora blob 배경이 아바타 텍스트 가독성에 영향을 줄 수 있음

## 단계 4 — 공간 UX 개선 아이디어

### 레퍼런스 분석

**Gather.town 패턴:**
- 아바타 주변 "인터랙션 반경" 원형 표시 (근처 진입 시 자동 비디오 활성)

**Figma FigJam 패턴:**

- 혼잡 시 zoom-out 하여 minimap 활용

### 개선안 3가지

---

#### 1. **[높음]** 빈 공간 탭-투-무브 (Click-to-Move)

**문제:** 모바일에서 52px 아바타를 드래그해서 이동하기 어려움. 화면 전체가 이동 가능한데, 오직 아바타를 잡아서 드래그해야 함.

**제안:** 캔버스 빈 공간 탭 → 내 아바타가 스프링 애니메이션으로 그 위치로 이동
- Gather.town의 핵심 UX 패턴
- 드래그는 세밀한 위치 조정용으로 유지, 탭은 빠른 이동용

**구현 위치:**
- `RoomPage.jsx`: `containerRef` div에 `onClick` 핸들러 추가 (단, 아바타/메뉴 클릭은 stopPropagation으로 제외)
- `useCanvasInteraction.js`: `handleCanvasClick(e)` 함수 추가 — 클릭 좌표 → % 변환 → `setMe` + `emit('move')`
- 예상 난이도: ⭐⭐ (낮음)

```jsx
// RoomPage.jsx containerRef div에 추가
onClick={(e) => {
  menu && setMenu(null);
  // 아바타/메뉴가 아닌 빈 공간 클릭 시 이동
  if (e.target === containerRef.current || e.target.closest('.dot-grid-dark')) {
    handleCanvasClick(e);
  }
}}
```

---

#### 2. **[높음]** 내 아바타 주변 "통화 반경" 링 시각화

**문제:** 현재 `ProximityAura`는 나(me) 중심에 상대방 색상의 원을 그림. 직관성이 낮음 — "이 안에 들어오면 가까운 것"임을 모름.

**제안:** 내 아바타 주변에 점선 원(dashed circle) 표시 — "이 안에 들어오면 연결됨" 시각화
- 아무도 근처 없으면 희미한 점선, 누군가 들어오면 색상 하이라이트
- FigJam의 "cursor zone" 개념

**구현 위치:**
- `ProximityAura.jsx`: me 주변 `MAX_DIST` 반경 SVG circle 추가
- 또는 `Avatar.jsx` isMe 아바타에 `ring` 애니메이션 추가

```jsx
// ProximityAura.jsx에 me 반경 링 추가
const hasNearby = users.some(u => {
  const dist = Math.hypot((u.x - me.x)/100*ww, (u.y - me.y)/100*wh);
  return dist < MAX_DIST;
});

<div
  className="absolute rounded-full pointer-events-none"
  style={{
    left: `${me.x}vw`,
    top: `${me.y}vh`,
    width: `${MAX_DIST * 2}px`,
    height: `${MAX_DIST * 2}px`,
    transform: 'translate(-50%, -50%)',
    border: `1.5px dashed rgba(${AURA_RGB[me.avatar]}, ${hasNearby ? 0.5 : 0.15})`,
    transition: 'border-color 0.4s ease',
  }}
/>
```
- 예상 난이도: ⭐ (매우 낮음)

---

#### 3. **[중간]** 혼잡 룸 가독성 — 아바타 스택 카운터 + 확장 토글

**문제:** 클러스터 팬아웃이 자동으로 퍼뜨리지만, 5명 이상 모이면 원이 커져서 화면 밖으로 나갈 수 있음. 또한 아바타 위 텍스트(9-10px)가 작아 혼잡 시 읽기 어려움.

**제안:**
1. 3명 이상 클러스터 시 "그룹 아이콘" + 숫자 badge로 접기 (FigJam grouping)
2. 클러스터 탭 시 펼쳐서 개별 인터랙션 가능
3. 또는 더 간단하게: 클러스터 내 아바타 hover 시 닉네임 pill을 더 크게 (12px) 표시

**구현 위치:**
- `useCanvasInteraction.js`: `clusterOffsets` 로직에 그룹 크기 정보 추가
- `RoomPage.jsx` 또는 새 `AvatarCluster.jsx` 컴포넌트
- 예상 난이도: ⭐⭐⭐ (중간 — 상태 관리 필요)
