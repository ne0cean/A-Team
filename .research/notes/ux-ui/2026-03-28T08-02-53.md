## 단계 1 — 2D 인터랙션 구현 방식

### 이동 로직
- **좌표계**: 0–100% (vw/vh 기반 퍼센트). `_calcDragPos`에서 `info.point` → rect 보정 → zoom 보정 → 5–95% clamping.
- **소켓 emit 스로틀**: 80ms (≈12.5fps). 네트워크 부하 적절히 억제.
- **내 아바타**: `x`/`y` transition은 `type:'tween', duration:0` → 즉각 반응 (입력 지연 없음 ✅)
- **다른 유저**: `SPRING_MOVE` (stiffness=180, damping=28) 으로 smooth 추종

### 근접 감지
- `maxDist = min(ww, wh) * 0.15` 범위 내 가장 가까운 유저 → `nearestUserId`
- `proximityScaleMap`: 가까울수록 최대 +15% scale (미묘하게 커짐)

### 클러스터 fan-out
- 5% 이내에 겹친 유저들을 원형으로 펼침 (`radius = 3.5 + n * 0.5`)
- 정렬 기준: ID 알파벳 순 (결정론적, 재렌더링 안정적)

### 더블탭
- `lastTapTime` ref로 350ms 이내 재탭 감지 → `ping_wave` emit

### 주목할 점
- `enableZoom=false`가 RoomPage 기본값 → pinch/wheel zoom 비활성화됨
- zoom state는 훅 내부에 항상 존재하지만 RoomPage에서 미사용 (낭비 아님, 경미)

## 단계 2 — Framer Motion 애니메이션 분석

### 현재 spring 값

| 상수 | stiffness | damping | mass | 용도 |
|---|---|---|---|---|
| `SPRING_ENTER` | 350 | 22 | 1.1 | 아바타 입장, opacity, rotate |
| `SPRING_MOVE` | 180 | 28 | 0.8 | 다른 유저 위치 추종 |
| `BOUNCE_HOVER` | 400 | 12 | — | 내 아바타 scale hover |

### 평가

**SPRING_ENTER (stiffness=350, damping=22)** — ✅ 적합
- 약한 오버슈트로 "툭 나타나는" 느낌. 빠르고 가볍다. Vibe Here의 캐릭터와 잘 맞음.

**SPRING_MOVE (stiffness=180, damping=28)** — ⚠️ 조건부 적합
- 낮은 stiffness → smooth하게 따라가지만 네트워크 지연(100ms+)에서 위치 emit이 드문드문 오면 "꼬리 흔들림" 현상 발생 가능.
- 개선 제안: `stiffness: 220, damping: 30`으로 약간 올리면 jitter 줄이면서 자연스러움 유지.

**BOUNCE_HOVER (stiffness=400, damping=12)** — ✅ 의도적 선택
- damping=12는 매우 통통 튀는 bounce. 장난기 있고 playful → Vibe Here 감성에 딱 맞음.

**아이들 bob** — ✅ 좋음
- `duration: 2.4s, y: [0, -5, 0]` → 느리고 부드러운 호흡 같은 움직임. 공간감 생성.
- `bobDelay`: socket ID 기반 stagger → 모든 아바타가 동시에 오르내리지 않아 자연스러움.

**EphemeralShout** — ✅ 좋음
- `times: [0, 0.08, 0.75, 1]` → 빠른 fade-in (8%), 오래 노출(67%), 부드러운 fade-out. 가독성 좋음.

**UserMenu 팝업** — ✅ 좋음
- `stiffness:300, damping:25` → 빠르고 깔끔한 팝업. 지연 없이 즉각 반응.

### 트랜지션 타이밍 제안

| 상황 | 현재 | 제안 | 이유 |
|---|---|---|---|
| 다른 유저 이동 | stiffness=180 | stiffness=220 | 고지연 환경에서 tail 줄이기 |
| 아이들 bob | y=[0,-5,0] | y=[0,-4,0] | 혼잡한 룸에서 시각적 진동 감소 |
| mood 변경 | duration=0.22 | duration=0.18 | 더 빠른 감정 전달 |

## 단계 3 — 접근성 & 모바일 감사

### aria-label 현황

| 요소 | 상태 | 비고 |
|---|---|---|
| 헤더 버튼 (복사/프라이버시/기분/shout/mic/나가기) | ✅ 있음 | |
| Avatar (다른 유저) | ✅ `role="button"` + `aria-label="${nickname}과 상호작용"` | |
| UserMenu "DM 보내기" 버튼 | ⚠️ 텍스트만, aria-label 없음 | 스크린리더엔 텍스트로 인식되므로 허용 범위 |
| UserMenu "리액션" 버튼 | ⚠️ 텍스트만 | 위와 동일 |
| ProximityAura `<div>` 요소들 | ✅ `pointer-events-none` → 접근성 불필요 | |
| ConnectionLine `<svg>` | ❌ `aria-hidden` 없음 | 장식 SVG에 aria-hidden 추가 필요 |
| EphemeralShout | ❌ `role="status"` 또는 `aria-live="polite"` 없음 | 스크린리더가 shout 감지 불가 |

### 키보드 내비게이션

- ✅ 글로벌 `button:focus-visible` ring 있음 (index.css)
- ✅ Escape로 모든 오버레이 닫기
- ✅ Avatar (다른 유저): onKeyDown Enter → 클릭 처리
- ❌ **UserMenu focus trap 없음**: 메뉴 열릴 때 자동 focus 이동 없음, Tab으로 닫기 어려움
- ❌ **MoodPicker/PrivacyModal**: AnimatePresence로 마운트되지만 focus trap 미확인 필요

### 모바일 터치 지원

- ✅ `touch-none` (내 아바타 드래그) / `touch-manipulation` (다른 유저)
- ✅ `navigator.vibrate(8)` 드래그 시작 햅틱 피드백
- ✅ 헤더 버튼 터치 영역 ~48px (px-4 py-3) → WCAG 44px 기준 충족
- ✅ 더블탭 350ms threshold → 모바일에서 적절
- ❌ **ConnectionLine 회전 미대응**: `window.innerWidth/Height` 직접 읽음, resize listener 없음 → 화면 회전 시 좌표 깨짐
- ❌ **롱프레스 없음**: 탭 1회 → 즉시 메뉴 오픈. 모바일에서 이동 중 실수 메뉴 오픈 빈번
- ❌ **pinch zoom 비활성화**: `enableZoom=false`. 혼잡한 룸에서 줌인 불가. 의도적 선택인지 미완성인지 불명
- ⚠️ `[@media(pointer:coarse)]:opacity-100` 상태 버블 — 모바일에서 항상 보이도록 처리됨 ✅

#### 개선안 1 — [높음] ConnectionLine 상대방 색상 반영
**현황**: ConnectionLine의 stroke가 항상 teal 고정 (`rgba(20,184,166,...)`)
**문제**: 여러 유저가 있을 때 "누구와 연결됐는지" 색으로 구별 불가
**제안**: `nearestUser.avatar` 기반 색상 lookup + SVG linearGradient로 me 색 → nearest 색 그라디언트

수정 파일: `client/src/components/ConnectionLine.jsx`
변경 내용:
```jsx
// 상단 color map 추가
const LINE_COLOR = {
  teal: '20,184,166', fuchsia: '192,38,211', blue: '59,130,246',
  rose: '244,63,94', amber: '245,158,11', indigo: '99,102,241',
};

// SVG 내부에 <defs><linearGradient> 추가, stroke를 url(#grad) 참조
// meColor = LINE_COLOR[me.avatar] ?? LINE_COLOR.teal
// nearestColor = LINE_COLOR[nearestUser.avatar] ?? LINE_COLOR.teal
```
예상 난이도: ★☆☆ (낮음, SVG defs 추가만)

#### 개선안 2 — [높음] 모바일 롱프레스 제스처로 메뉴 오픈
**현황**: 탭 1회 → 즉시 UserMenu 오픈
**문제**: 이동 중 아바타에 손가락이 닿으면 의도치 않게 메뉴 열림
**제안**: `pointerdown` → 200ms 타이머 → `pointerup` 취소 없으면 롱프레스로 메뉴 오픈. 탭은 더블탭 or ping에만 반응.

수정 파일: `client/src/components/Avatar.jsx`
변경 내용:
```jsx
// onClick 대신 onPointerDown/Up + timer 패턴
// hasDragged.current가 false이고 200ms 내 pointerup 없으면 longPress
// 현재 onClick의 역할을 longPress로 이전 (다른 유저 메뉴)
```
예상 난이도: ★★☆ (중간, touch/pointer 이벤트 핸들링 재설계)

#### 개선안 3 — [중간] EphemeralShout aria-live 영역 추가
**현황**: Shout 말풍선이 화면에 렌더링되지만 스크린리더에게 전달 안 됨
**문제**: 시각 장애 사용자가 shout 내용 인지 불가
**제안**: RoomPage에 `aria-live="polite"` 영역 추가, shout 수신 시 텍스트 삽입

수정 파일: `client/src/pages/RoomPage.jsx`
변경 내용:
```jsx
// div[aria-live="polite"][aria-atomic="true"][className="sr-only"] 추가
// shouts state 변경 시 최신 shout.text를 해당 div에 반영
```
예상 난이도: ★☆☆ (낮음)

#### 개선안 4 — [중간] ConnectionLine resize 대응
**현황**: `window.innerWidth/Height` 직접 읽음, 컴포넌트 내 resize listener 없음
**문제**: 모바일 화면 회전 시 연결선 좌표 틀어짐
**제안**: ProximityAura의 `useWindowSize` 훅 공유 또는 ConnectionLine 내 동일 훅 사용

수정 파일: `client/src/components/ConnectionLine.jsx`
변경 내용: ProximityAura에 있는 `useWindowSize` 훅을 별도 파일로 분리 후 ConnectionLine에서 import
예상 난이도: ★☆☆ (낮음, 훅 추출)

---

#### 개선안 5 — [낮음] 장식 SVG에 aria-hidden 추가
**현황**: ConnectionLine의 `<svg>`에 `aria-hidden` 없음

**제안**: `<svg aria-hidden="true">` 추가

수정 파일: `client/src/components/ConnectionLine.jsx`
