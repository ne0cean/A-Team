## 단계 1 — 2D 인터랙션 구현 방식

### 이동 로직
- **위치 저장 방식**: 0~100% 범위 (x, y) — viewport 독립적. `vw`/`vh`로 렌더링.
- **드래그 스로틀**: `lastDragEmit.current` 비교로 80ms 간격으로만 `socket.emit('move')` 발송. 네트워크 부하 낮음.
- **내 아바타 이동**: Framer Motion `drag` 속성 + `dragMomentum: false` → 관성 없이 즉각 반응.
- **위치 최종 확정**: `onDragEnd`에서만 `setMe` 상태 갱신 → 드래그 도중 리렌더 최소화.

### 더블탭 감지
- `lastTapTime.current` ref로 350ms 이내 두 번 클릭 = 더블탭 → `ping_wave` 발송.
- 모바일에서도 동작하는 구조이나, 터치 드래그와 탭 구분이 `hasDragged` ref (3px threshold)에 의존.

### 클러스터 팬아웃 (clusterOffsets)
- THRESHOLD=5 (% 단위, ~1920px 기준 약 96px) — 겹친 아바타들을 원형으로 분산.
- 반경: `3.5 + cluster.length * 0.5` → 유저 수 비례 확장.
- y축에 `* 0.7` 보정 — 타원형 팬아웃, 공간감 있음.

### 근접 감지
- `maxDist = min(vw, vh) * 0.15` 범위 내 가장 가까운 유저 = nearestUserId.
- `proximityScaleMap`: 거리 기반 아바타 크기 1.0~1.15 사이 스케일.

## 단계 2 — 컴포넌트 애니메이션 분석

### Framer Motion Spring 값 비교

| 상수 | stiffness | damping | mass | 용도 |
|------|-----------|---------|------|------|
| `SPRING_ENTER` | 350 | 22 | 1.1 | 입장/퇴장, opacity, scale |
| `SPRING_MOVE` | 180 | 28 | 0.8 | 다른 유저 이동 (x, y) |
| `BOUNCE_HOVER` | 400 | 12 | — | 호버 scale |

### "가볍고 빠른" UX 적합성 평가

**✓ 잘 된 것:**
- 내 아바타 이동 = `type: 'tween', duration: 0` → 즉각 반응, 지연 없음.
- `BOUNCE_HOVER` (damping:12) = underdamped → 탄력 있는 호버 피드백, 경쾌한 느낌.
- idle bob: 2.4초 사이클 + `bobDelay` stagger로 자연스러운 생동감.
- 기분 변경: remount + `{ scale: 1.15, opacity: 0 } → { scale: 1, opacity: 1 }` 0.22초, 빠르고 깔끔.

**△ 개선 여지:**
- `SPRING_MOVE` (stiffness:180)은 부드럽지만 다소 무거운 느낌. 빠른 이동 시 "쫓아가는" 지연이 체감될 수 있음. **250~280 권장**.
- `SPRING_ENTER` (mass:1.1)은 입장 애니메이션에 약간의 무게감 추가 → 입장이 "탁" 나타나는 느낌보다 "쭉" 들어오는 느낌. `mass: 0.8`로 낮추면 더 경쾌.
- 연결 상태 배너의 spring (stiffness:300, damping:28) — 적절함.

### 트랜지션 타이밍 제안 (우선순위순)
1. `SPRING_MOVE.stiffness` 180 → 260: 다른 유저 이동 반응성 +44%, 부드러움 유지.
2. `SPRING_ENTER.mass` 1.1 → 0.8: 입장 애니메이션 15% 빠름.
3. idle bob `duration` 2.4 → 3.0: 현재 약간 빠름, 더 여유로운 ambience.

## 단계 3 — 접근성 & 모바일 감사

### aria-label 현황
| 요소 | 상태 |
|------|------|
| 초대 코드 복사 버튼 | ✓ `aria-label="초대 코드 복사"` |
| 프라이버시 버튼 | ✓ `aria-label="프라이버시 설정 변경"` |
| 기분 버튼 | ✓ `aria-label="기분 설정"` |
| Shout 버튼 | ✓ `aria-label="말풍선 외치기"` |
| 마이크 버튼 | ✓ 동적 aria-label (상태 반영) |
| 나가기 버튼 | ✓ `aria-label="룸에서 나가기"` |
| 다른 유저 아바타 | ✓ `aria-label="{nickname}과 상호작용"` |
| 내 아바타 | ✗ **aria-label 없음** — drag 영역이 역할 불명확 |
| ProximityAura div | ✓ `pointer-events-none` + 장식 요소, aria 불필요 |
| 사용자 수 pill | ✗ **aria-label 없음** — 스크린리더 "16 명" 텍스트만 읽힘 |

### 키보드 내비게이션
- 다른 유저 아바타: `role="button"`, `tabIndex={0}`, `onKeyDown` → Enter 키 동작 ✓
- 내 아바타: `tabIndex` 없음 → 키보드로 접근 불가. 드래그 특성상 제한적이나 **Enter = ping_wave** 같은 단축키 바인딩 고려 가능.
- 모달 Escape 닫기: `window.addEventListener('keydown')` ✓
- `focus-visible:ring` 스타일: 헤더 버튼 전체 적용됨 ✓

### 모바일 터치 지원
| 기능 | 상태 | 비고 |
|------|------|------|
| 아바타 드래그 | ✓ | `touch-none` (isMe), Framer Motion drag |
| 아바타 탭 | ✓ | `touch-manipulation` |
| 더블탭 ping | ✓ | 350ms threshold |
| 핀치 줌 | **✗ RoomPage에서 비활성** | `enableZoom=false` (기본값), SpacePage만 활성 |
| 가로모드 레이아웃 | △ | 헤더 버튼 overflow 가능 (gap:3, 8개 버튼) |
| Status hover bubble | △ | `[@media(pointer:coarse)]:opacity-100` — 모바일에서 항상 표시 → 공간 차지 |
| 터치 haptic | ✓ | `navigator.vibrate(8)` on drag start |

**주요 모바일 이슈:**
1. 헤더 우측 버튼 8개 (Privacy, Mood, Shout, QR, 유저수, Mic, Leave) — iPhone SE(375px)에서 overflow/wrapping 가능성.
2. 모바일에서 아바타 클릭 메뉴(UserMenu)가 `position: { x: user.x, y: user.y }` 기반 → 화면 가장자리 아바타 클릭 시 메뉴 잘릴 수 있음.
3. Shout 인풋 `border border-slate-200` + `text-slate-800` — glass-dark 배경 위 어두운 텍스트 → 가독성 문제 (다크 배경에 다크 텍스트).

## 단계 4 — 공간 UX 개선 아이디어

### 레퍼런스 패턴 분석
- **Gather.town**: 타일 기반 이동 (W/A/S/D 또는 tap-to-move), 근접 시 비디오 자동 활성화.
- **Figma FigJam**: 드래그로 자유 이동, 커서 위치 실시간 공유, multiplayer 인디케이터.
- **Discord Stage**: 화자 하이라이트 + 청중 시각화.

### 근접 감지 시각적 표시 개선

**현재 상태**: ProximityAura는 `opacity`와 `size` 변화만 있고 CSS 트랜지션 없음 → 위치가 바뀌면 즉시 점프.

**제안**:
- `transition: 'opacity 0.3s ease, width 0.3s ease, height 0.3s ease'` 추가
- nearestUser에게 Framer Motion `animate={{ scale: [1, 1.08, 1] }}` pulse 추가 (현재는 CSS `animate-pulse` ring만)
- 거리에 따라 색상 변화: 매우 가까움(50px 이내) → 따뜻한 amber, 근접(150px) → teal, 그 외 → 없음

### 혼잡한 룸 가독성 개선

**현재 상태**: clusterOffset이 팬아웃하지만 닉네임 pill (max-w-[72px] truncate)이 여전히 겹칠 수 있음.

**제안**:
- 유저 10명 이상 시 닉네임 pill → 호버(또는 탭)에서만 표시 (`opacity: 0 → 1` transition)
- `[@media(pointer:coarse)]:opacity-100` 조건을 유저 수 threshold로 조정: 5명 이하 = 항상 표시, 6명+ = hover-only

### 3가지 구체적 UX 개선안

---

**[1] Tap-to-Move 이동 패턴 추가 (높음)**

현재는 아바타를 직접 드래그해야만 이동 가능. 모바일에서 드래그 정밀도 낮음.

- **수정 파일**: `RoomPage.jsx` (캔버스 onClick) + `useCanvasInteraction.js`
- **구현 방법**: 캔버스 빈 공간 클릭 시 해당 좌표로 내 아바타 이동 emit.
  ```
  containerRef onClick → (e.target === container 확인) →
  클릭 좌표 → % 변환 → setMe + emit('move')
  ```
- **참고 패턴**: Gather.town 방식, FigJam 커서 이동
- **난이도**: 낮음 (30~50줄)
- **UX 기대효과**: 모바일 진입 장벽 최소화, 클릭으로 공간 탐색

---

**[2] 헤더 버튼 오버플로우 → 하단 고정 액션 바 (모바일) (중간)**

현재 헤더 우측 8개 버튼이 좁은 화면에서 깨질 위험.

- **수정 파일**: `RoomPage.jsx` (헤더 div)
- **구현 방법**: `md:flex hidden` / `md:hidden flex` 분기로 모바일 전용 하단 바 추가.
  - 모바일: 하단 fixed bar에 핵심 4개만 (Mood, Shout, Mic, Leave)
  - 나머지(QR, Privacy, 유저 수)는 "더보기" 버튼 뒤로
- **난이도**: 중간 (CSS 분기 + AnimatePresence 모달)
- **UX 기대효과**: 모바일 엄지손가락 영역 최적화, 인터랙션 밀도 감소

---

**[3] ProximityAura 애니메이션 + 근접 텍스트 힌트 (낮음)**

현재 오라가 CSS transition 없이 즉시 변화. 가장 가까운 유저에 대한 시각적 힌트 부족.

- **수정 파일**: `ProximityAura.jsx`, `Avatar.jsx`
- **구현 방법**:
  1. ProximityAura의 div에 `transition: 'opacity 0.4s, width 0.4s, height 0.4s'` 추가
  2. nearestUser 아바타 아래에 작은 "가까이 있어요" 툴팁 (거리 < maxDist * 0.3일 때만)
  3. Framer Motion으로 오라 pulse: `animate={{ scale: [1, 1.04, 1] }}` 2초 반복
- **난이도**: 낮음 (10~20줄 추가)
- **UX 기대효과**: 공간 인식 명확화, "언제 말 걸 수 있는지" 직관적 시각화
