## 5. DR 시나리오별 플레이북

### A. 호스팅 제공자 전면 장애
1. 상태 페이지 확인 → ETA 파악
2. ETA > 4시간 → 대체 서비스 배포
3. DNS 전환 (Cloudflare에서 즉시)
4. 고객 상태 페이지 업데이트

### B. DB 손상/손실
1. 최신 백업 확인 (RPO 내인지)
2. 새 인스턴스에 복원
3. 앱 DB 연결 문자열 변경
4. 데이터 무결성 검증

### C. 도메인/DNS 장애
1. 레지스트라 로그인 확인
2. 네임서버 변경 (최후 수단)
3. IP 직접 접속 안내 (임시)

### D. 본인 불능 (질병/사고)
1. 비상 접근 권한 문서 (가족/신뢰인)
2. 자동 갱신 설정 확인 (도메인, 호스팅, Stripe)
3. "X일간 응답 없으면" 자동 공지 설정
