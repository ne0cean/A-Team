# Prompt Coaching — 5가지 실패 유형 상세

> `/end` Step 6.7에서 참조. 세션 종료 시 프롬프트 분석용.

## 1. 의도 오해 (Claude가 엉뚱한 작업 수행)

| 패턴 | 예시 | 문제 |
|------|------|------|
| 암묵적 기대 | "개선해줘" | 무엇을 어떻게? |
| 다의적 표현 | "이거", "저거" | 지시대상 불명확 |
| 추상적 목표 | "좋게 만들어" | 기준 없음 |
| 비유/은유 | "깔끔하게" | 주관적 해석 |
| 생략된 주어 | "추가해" | 뭘 어디에? |

**공식**: `[대상] + [행동] + [기대결과] + [제약]`

## 2. 스코프 폭주 (안 해도 될 작업까지 수행)

| 패턴 | 예시 | 결과 |
|------|------|------|
| 범위 미지정 | "auth 개선해" | 전체 리팩토링 |
| 제약 미명시 | "OAuth 추가" | 기존 코드 변경 |
| 열린 요청 | "할 수 있는 것 다 해줘" | 과잉 작업 |

**공식**: `[작업] + [경계] + [제외항목]`
**키워드**: "~만", "~에서만", "~는 건드리지 마"

## 3. 결과물 불일치 (원하는 형태가 아님)

| 패턴 | 예시 | 결과 |
|------|------|------|
| 형식 미지정 | "분석해줘" | 텍스트 vs 표 vs JSON |
| 깊이 미지정 | "설명해줘" | 한 줄 vs 상세 |
| 스타일 미지정 | "코드 작성해" | 함수형 vs 클래스 |

**공식**: `[작업] + [출력형식] + [품질기준]`
**키워드**: "표로", "JSON으로", "~줄 이내", "테스트 포함"

## 4. 컨텍스트 단절 (기존 결정/패턴 무시)

| 패턴 | 예시 | 결과 |
|------|------|------|
| 히스토리 무시 | 이미 논의한 내용 재설명 | 토큰 낭비 |
| 기존 결정 무참조 | "어떻게 할까?" | 재논의 |

**공식**: `[컨텍스트 참조] + [작업]`
**키워드**: "CURRENT.md 참고", "위에서 말한", "@파일명"

## 5. 재작업 루프 (같은 작업 반복 수정)

| 패턴 | 예시 | 결과 |
|------|------|------|
| 검증 기준 없음 | "만들어줘" | 완료 판단 불가 |
| 종료 조건 없음 | "개선해줘" | 멈출 시점 모름 |

**공식**: `[작업] + [완료조건] + [검증방법]`
**키워드**: "~하면 완료", "테스트 통과", "빌드 성공"

## 출력 템플릿

```
📝 프롬프트 코칭 리포트
━━━━━━━━━━━━━━━━━━━━━
📊 이번 세션: 프롬프트 N개 | 재작업 M회 | 명확화 K회
🔴 실패 유형: (해당 유형 + 예시)
📝 개선 예시: Before → After
💡 다음 세션: □ 대상+행동+결과+제약 □ 경계 키워드 □ 완료 조건
```
