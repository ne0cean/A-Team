# 모니터링 약속 금지 — 스케줄링 강제

Claude는 세션 리셋 후 능동적 모니터링이 불가능하다.
"지켜보겠다", "추이를 보자", "나중에 확인하자"는 실행되지 않는 거짓 약속이다.

## 강제 규칙

### 1. 금지 표현
다음 표현을 사용할 수 없다:
- "지켜보겠다", "추이를 보자", "모니터링하겠다"
- "다음에 확인", "나중에 검토", "시간을 두고 보자"
- "며칠 후 재평가", "한 달 뒤 점검"

### 2. 대체 행동 (택일)

**A. 즉시 자동화**: vitest/cron/hook으로 검증 인프라 구축
```
"npm test에 검증 테스트를 추가했으므로 매 커밋마다 자동 검증됩니다."
```

**B. 스케줄 등록**: `.context/scheduled-reviews.json`에 항목 추가
```json
{
  "id": "고유-슬러그",
  "due": "YYYY-MM-DD",
  "title": "검토 내용 1줄",
  "context": "왜 이 날짜에 검토가 필요한지",
  "source_commit": "해당 커밋 해시",
  "created": "오늘 날짜",
  "status": "pending"
}
```
등록 후 사용자에게 알림: "YYYY-MM-DD에 자동 리마인드되도록 등록했습니다."

**C. 솔직한 고백**: 자동화도 스케줄도 불가능하면 명시
```
"이 항목은 자동 검증 수단이 없고, 스케줄 등록도 적절하지 않습니다.
수동 확인이 필요하며, 제가 자동으로 리마인드할 수 없습니다."
```

### 3. 스케줄 라이프사이클

| 단계 | 행동 |
|------|------|
| 등록 | scheduled-reviews.json에 추가, status: "pending" |
| 도래 | vibe/pickup Step 0.75에서 자동 표시 |
| 처리 | 검토 후 status → "done", 필요 시 새 항목 등록 |
| 미처리 | 매 세션마다 초과 일수와 함께 반복 표시 |

### 4. 위반 감지

이 규칙 위반 시 (모니터링 약속 + 스케줄 미등록):
- 사용자가 즉시 지적 가능
- 다음 /review, /ship에서 adversarial 에이전트가 "빈 약속" 패턴으로 감지
