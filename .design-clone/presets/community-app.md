# Preset: Community App — 로컬 커뮤니티 / 소셜 앱 랜딩

> flair 같은 위치기반 소셜 앱의 초기 랜딩. "친구가 만든 앱" 느낌.

## 대표 레퍼런스 패턴

| 출처 유형 | 예시 | 왜 효과적인가 |
|----------|------|-------------|
| 대학생 사이드프로젝트 | 캠퍼스 앱들 | 타깃과 만든 사람이 동일 |
| Product Hunt Day 1 | 초기 소셜 앱 | "방금 만들었어요" 에너지 |
| 카페 게시판 전단지 디지털화 | 동네 커뮤니티 | 신뢰 + 로컬 |
| Discord 서버 초대 페이지 | 커뮤니티 기반 | 참여 장벽 최소 |

## 해부 결과 (공통 패턴)

```yaml
font:
  heading: "시스템 폰트 또는 1개 Google Font (Poppins, Noto Sans KR)"
  body: "시스템 폰트"
  sizes: [28px, 16px, 14px]
  특징: "제목만 살짝 큼. 나머지 평범."

color:
  palette: ["#111", "#fff", 브랜드색 1개]
  accent: "앱 성격 반영 (소셜=파랑/초록, 데이팅=핑크/보라)"
  특징: |
    - 배경 흰색 또는 아주 연한 색
    - 다크모드 미지원 (초기니까)
    - 브랜드색이 CTA 버튼에만 집중

layout:
  구조: "1단 세로, 모바일 우선"
  전형적 순서:
    1: "앱 이름 + 한 줄 설명"
    2: "폰 목업 스크린샷 1장 (있으면)"
    3: "핵심 기능 3개 (짧은 리스트, 카드 아님)"
    4: "앱스토어 링크 또는 대기자 폼"
    5: "인스타/트위터 링크"
  특징: |
    - 폰 목업이 있으면 시선 끌기 효과적
    - 없으면 텍스트만으로도 OK
    - 기능 설명이 2줄 이하 (장황하면 AI 느낌)

copy_tone:
  스타일: "친근, 또래 말투"
  특징: |
    - "주변 사람들과 바로 연결돼요"
    - "앱 깔고 QR만 찍으면 끝"
    - 기술 용어 금지 (WiFi BSSID, geofencing 등)
    - "만드는 중이에요" 솔직함 OK
    - "00명이 기다리고 있어요" (실제 숫자, 작아도 OK)

imagery:
  유형: "앱 스크린샷 또는 폰 목업 1장"
  특징: |
    - 스크린샷이 완벽하지 않아도 됨 (개발 중 화면 OK)
    - 사람 사진보다 앱 화면이 중요
    - 실제 사용 장면 (카페에서 폰 들고 있는 사진) 1장이면 최고
```

## Intentional Imperfections

```yaml
필수 반영:
  - 앱스토어 링크 미완성 ("곧 출시" 또는 TestFlight 링크)
  - "현재 성수/강남 지역만 가능" 같은 한정 범위
  - 폰 목업 안의 스크린샷이 100% 최신 아님
  - 대기자 수가 현실적 (12명, 47명 — 10,000명 아님)
  - SNS 팔로워 수 노출하지 않음 (적으니까)
  - "피드백 주세요" 구글폼 링크
  - 팀 소개 없음 또는 "만든 사람: @핸들" 한 줄

선택 반영:
  - beta 배지 또는 "v0.1" 표시
  - "아직 버그 있을 수 있어요" 솔직 문구
  - 개인 블로그/노션에서 만든 사람 소개 링크
```

## Anti-Pattern

```yaml
절대 하지 말 것:
  - "Join 10,000+ users" (거짓 social proof)
  - 투자 유치 금액 표시
  - "Backed by Y Combinator" 배지
  - 3D 폰 목업 (Figma 커뮤니티 템플릿 냄새)
  - 앱스토어 + 플레이스토어 뱃지 나란히 (아직 없으면서)
  - 비디오 히어로 (YouTube 임베드)
  - 팀 사진 그리드 (링크드인 느낌)
  - pricing table
```
