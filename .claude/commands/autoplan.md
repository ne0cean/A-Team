# /autoplan — 자동 계획 검토 파이프라인

CEO → 디자인 → 엔지니어링 검토를 순서대로 실행하고, 기계적 판단은 6가지 원칙으로 자동 결정한다.
사람의 판단이 필요한 "취향 결정"만 마지막 게이트에서 묻는다.

## 언제 사용하나
- 구현 시작 전 계획/PRD/설계 문서를 검토할 때
- `/office-hours` 결과물을 검증할 때
- 큰 기능을 시작하기 전 리스크를 미리 잡을 때

---

## Phase 0: 준비

```bash
# 검토할 계획 파일 확인 (우선순위 순)
# 1. 명시적으로 지정된 파일
# 2. .context/에서 최근 수정된 계획 파일
# 3. ~/.gstack/projects/에서 최근 설계 문서
```

읽을 컨텍스트:
- `CLAUDE.md` — 프로젝트 규칙
- `.context/CURRENT.md` — 현재 상태
- 계획 파일 전체

**자동 결정 원칙 (6가지)** — 판단 가능한 것은 자동 결정, 아닌 것만 Phase 4에서 질문:
1. 완전성 우선 — 부분 구현 < 완전 구현
2. 보이는 건 고친다 — 1일 미만 비용이면 함께 처리
3. 실용적 선택 — 결과 동일하면 단순한 옵션
4. DRY 강제 — 기존 코드 재사용
5. 명시적 > 영리한
6. 행동 편향 — 계획보다 실행

## Phase 1: CEO 검토 (전략 & 범위)

**검토 항목:**
- [ ] 핵심 전제가 현실적인가? (수요, 실현 가능성)
- [ ] 범위가 명확히 정의되어 있는가?
- [ ] "NOT in scope" 섹션이 있는가?
- [ ] 실패 모드와 복구 계획이 있는가?
- [ ] 목표 상태(dream state)와 현재 갭이 명확한가?

**출력:**
```
## CEO 검토 결과
- 전제 도전: [검증/반박]
- 범위 결정: [자동 결정 + 원칙 번호]
- NOT in scope: [목록]
- 실패 모드 테이블: [리스크 | 확률 | 영향 | 대응]
```

**게이트**: 전제 중 자동 결정 불가한 것 → AskUserQuestion (배치, 한 번에)

---

## Phase 2: 디자인 검토 (UI 있는 경우만)

UI 관련 파일/언급이 계획에 없으면 자동 스킵.

**검토 항목 (7가지 차원):**
1. 사용자 흐름 — 핵심 경로가 명확한가?
2. 정보 구조 — 계층이 논리적인가?
3. 컴포넌트 재사용 — 기존 컴포넌트 활용하는가?
4. 반응형 — 모바일 고려되었는가?
5. 접근성 — ARIA, 키보드 내비게이션
6. 성능 — 렌더링 전략
7. 일관성 — 기존 디자인 시스템과 일치

기계적 수정 → 자동 결정, 취향 결정 → Phase 4 게이트로

---

## Phase 3: 엔지니어링 검토

**검토 항목:**
- [ ] 아키텍처 다이어그램 (ASCII) 생성
- [ ] 테스트 커버리지 다이어그램 — 각 코드 경로에 테스트 있는가?
- [ ] 성능 병목 예상 지점
- [ ] 보안 고려사항 (인증, 입력 검증, 데이터 분류)
- [ ] 의존성 리스크 (외부 서비스, 패키지)
- [ ] TODOS.md 업데이트 — 지연 항목 기록

**필수 산출물:**
```
아키텍처 다이어그램 (ASCII)
테스트 커버리지 맵
실패 모드 레지스트리
```

---

## Phase 4: 최종 승인 게이트

수집된 취향 결정들을 한 번에 제시:

```
다음 사항에 대한 결정이 필요합니다:

1. [취향 결정 A] — 선택지: X | Y
2. [취향 결정 B] — 선택지: X | Y

나머지는 자동 결정했습니다. [Decision Audit Trail 참조]
```

승인 시 → 검토 로그를 계획 파일에 기록 후 `status: DONE`
거부/수정 시 → 해당 Phase 재실행

---

## Decision Audit Trail

모든 자동 결정을 기록:
```
[원칙 1] 완전성: OAuth 구현 시 Google만 아닌 GitHub도 포함 → 완전 구현 선택
[원칙 4] DRY: 새 HTTP 클라이언트 만들지 않고 기존 axios 인스턴스 재사용
```

## 원칙
- **전체 깊이 필수** — 요약이 아닌 실제 코드/계획 분석
- **순차 실행** — CEO → 디자인 → 엔지니어링 순서 변경 불가
- **절대 중단 금지** — 사용자가 `/autoplan`을 선택했다는 것은 전체 실행 의사
