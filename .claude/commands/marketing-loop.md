# /marketing-loop — 주간 자가 개선 루프

**용도**: 주 1회 실행. 성과 분석 → 인사이트 → 프롬프트 개선 → 다음 주 전략. 마케팅 시스템의 자가 개선 엔진.

## 실행 흐름

### Step 1: 주간 성과 수집

`/marketing-analytics --period 7d --compare` 실행:
- 지난 7일 전 플랫폼 성과
- 이전 주 대비 증감
- 상위/하위 콘텐츠 목록

### Step 2: 패턴 분석 (Claude)

```
분석 질문:
1. 어떤 토픽이 가장 높은 참여율을 기록했는가?
2. 어떤 포맷이 (Twitter/LinkedIn/이메일) 이번 주 최고였는가?
3. 어떤 훅 패턴이 클릭을 만들었는가?
4. 어떤 포스팅 시간이 효과적이었는가?
5. AI 냄새가 감지된 콘텐츠가 있었는가? (낮은 참여율 기반 가설)
```

### Step 3: 프롬프트 라이브러리 업데이트 제안

성과 데이터 기반 제안 생성:
```markdown
## 이번 주 프롬프트 개선 제안

### 효과 있었던 것 (강화)
- blog.md: "{특정 패턴}" → 더 자주 사용 권장
- social-twitter.md: 훅 패턴 "{패턴}" 추가

### 효과 없었던 것 (제거/수정)
- social-linkedin.md: "{패턴}" → "{개선안}으로 교체"

적용: [y/N/일부만]
```

승인 시 → `prompts/` 파일 자동 업데이트 + git commit

### Step 4: 다음 주 콘텐츠 전략

```markdown
## 다음 주 콘텐츠 계획 (AI 제안)

우선순위 토픽 (성과 데이터 기반):
1. "{토픽}" — 이유: {이번 주 관련 토픽 성과}
2. "{토픽}" — 이유: {트렌드 + 검색 데이터}
3. "{토픽}" — 이유: {오디언스 질문 패턴}

추천 포맷 믹스:
- 블로그 2개 (월, 수)
- 소셜 전용 3개 (화, 목, 금)
- 이메일 1개 (목 오전)

승인하시겠습니까? [y/N/수정]
```

### Step 5: 주간 리포트 저장

```
content/analytics/
  └── YYYY-WW-weekly-report.md  (주차별)
      prompts-changelog.md      (프롬프트 변경 이력)
```

### Step 6: 다음 루프 예약

완료 후:
```
다음 /marketing-loop 실행: 7일 후 (YYYY-MM-DD)
수동 실행 또는 /schedule 로 자동화 가능
```

## 자동화 옵션

```bash
# 매주 월요일 오전 9시 자동 실행
/schedule --cron "0 9 * * 1" --command "marketing-loop"
```

## 루프 철학

이 루프가 없으면 마케팅 시스템은 정체됩니다.
Tatsuya Mizuno: "Your prompts are more valuable than any individual output."

매주 프롬프트가 개선되면:
- 4주 후: 콘텐츠 품질 30%+ 향상 (데이터 기반)
- 8주 후: 플랫폼 알고리즘 최적화
- 12주 후: 완전히 자신만의 브랜드 음성 확립
