# /mece-gap -- 아이디어 사이 미탐색 영역 발굴

전제: `memory/idea-registry.md` 존재. 없으면 안내 후 종료.

**핵심**: 카테고리 커버리지가 아니라, **개별 아이디어들 사이에서 아직 탐색하지 않은 조합/영역/관점**을 찾는다.

## Step 0 -- 프레임워크 로드

```bash
cat ~/Projects/a-team/cortex/thinking-toolkit.md | grep -A 200 "## Ideation Frameworks"
```

9프레임(F1-F9)을 로드한다. 이것이 탐색 렌즈.

## Step 1 -- 레지스트리 읽기 + 기존 아이디어 공간 파악

`memory/idea-registry.md` 전체를 읽고:
- 현재 아이디어의 **핵심 축** 추출 (예: 주체, 대상, 장소, 시간, 상호작용 방식, 정보 흐름)
- 축은 프로젝트마다 다르므로 레지스트리 내용에서 귀납적으로 도출

## Step 2 -- F1 형태학적 분석 (빈 셀 찾기)

Step 1에서 추출한 축들로 매트릭스 구성:

1. **축 정의** (레지스트리에서 귀납):
   - 예: 주체(사용자/AI/사물/장소) x 행동(보기/보내기/모으기/경쟁) x 맥락(혼자/1:1/그룹/전체)
   - 프로젝트별로 다름 — 기존 아이디어들의 패턴에서 추출

2. **기존 아이디어 매핑**: 각 아이디어가 매트릭스의 어느 셀에 해당하는지 표시

3. **빈 셀 식별**: 아이디어가 하나도 없는 조합 = 미탐색 기회
   ```
   | 주체\행동 | 보기 | 보내기 | 모으기 | 경쟁 |
   |----------|------|--------|--------|------|
   | 사용자   | B01,B03 | B10 | D01 | D03 |
   | AI/NPC   | E06 | ? | ? | ? |
   | 사물     | E04 | ? | ? | ? |
   | 장소     | A05 | ? | E17 | ? |
   ```
   `?` = 빈 셀 = 미탐색

4. **흥미로운 빈 셀 선별**: 모든 빈 셀이 기회는 아님. 다음 기준으로 필터:
   - 코어 루프 강화 가능성
   - 킬러 hook 후보 가능성
   - 기존 확정 아이디어와의 시너지

## Step 3 -- F2-F9 블라인드스팟 스캔

나머지 8프레임으로 기존 아이디어 전체를 훑으며 놓친 관점 탐지:

| 프레임 | 질문 | 방법 |
|--------|------|------|
| **F2 뒤집기+SCAMPER** | 확정 아이디어에 S/C/A/M/P/E/R 적용 시 새 변형? | 확정 상태 아이디어 각각에 7변환 시도 |
| **F3 행동 전수조사** | 레지스트리의 씬(N 카테고리 등)에서 디지털화 안 된 행동? | 씬별 미시행동 나열 → 기존 아이디어와 대조 |
| **F4 유추** | 타 산업 성공 패턴 중 이식 안 한 것? | "X의 Y를 여기 적용하면?" |
| **F5 기술 강제** | 최신 기술이 여는 문 중 탐색 안 한 것? | BLE/LLM/UWB/공간오디오 등 x 기존 아이디어 |
| **F6 경쟁 공백** | 경쟁사 전체가 안 하는 것 중 레지스트리에 없는 것? | 경쟁사 리서치 대조 |
| **F7 인간관점** | 의식주+오감 중 커버 안 된 영역? | 오감 x 기존 아이디어 매핑 |
| **F8 제약 놀이** | 현재 제약 제거 시 가능해지는 것 중 미탐색? | "X 제약 없으면?" |
| **F9 모순 해결** | 아직 풀리지 않은 모순 중 레지스트리에 해결책 없는 것? | 기존 모순 목록 대조 |

각 프레임당 **1-3개 갭**만 추출 (과다 금지).

## Step 4 -- 이전 대비 변화

```bash
LAST_MECE=$(ls memory/mece-ideation-gap-*.md 2>/dev/null | sort | tail -1)
```

이전 파일 있으면:
- 이전에 발견한 갭 중 닫힌 것 (새 아이디어로 채워진 것)
- 새로 발견된 갭
- 레지스트리 성장 추이 (아이디어 수 변화)

## 원칙

- **빈 셀 ≠ 기회**: 흥미로운 빈 셀만 선별. 의미 없는 조합은 버린다.
- **프레임당 3개 이하**: 과다 발산 방지. 날카롭게.
- **기존 아이디어 기반**: 새 아이디어를 만드는 게 아니라, 기존 사이의 빈틈을 찾는 것.
- **축은 귀납적**: 프로젝트마다 다른 축. 레지스트리 내용에서 추출.
