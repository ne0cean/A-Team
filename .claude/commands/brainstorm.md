# /brainstorm — 루프 안의 루프 브레인스토밍 모듈

> 파이프라인 단계가 아니라 **언제든 꽂는 터보**.
> `/office-hours`, `/pretotype`, `/prd` 등 어느 단계 중에서도 호출 가능.
> 호출 → 3단계 병렬 실행 → 결과 수령 → 원래 단계로 복귀.

## Step -1 — Ideation Frameworks 로드 (필수)

```bash
cat ~/Projects/a-team/cortex/thinking-toolkit.md | grep -A 200 "## Ideation Frameworks"
```

8프레임(F1-F8)을 읽고, EXPAND 단계에서 적절한 프레임을 선택 적용한다.

---

## 구조: 3중 루프

```
외부 루프 (days/weeks):  office-hours → pretotype → prd → build → validate
                                 ↕               ↕          ↕
중간 루프 (수분~수시간):       /brainstorm 호출 (어느 단계에서든)
                                          ↓
내부 루프 (한 호출 안에서):   EXPAND → ATTACK → SYNTHESIZE
                              (병렬)    (병렬)    (Judge 통합)
```

---

## Step 0 — 컨텍스트 수집

`$ARGUMENTS`가 있으면 바로 사용. 없으면 한 줄 질문:
> "지금 어떤 아이디어 또는 단계에서 브레인스토밍하고 싶으신가요?"

라운드 번호 자동 감지:
```bash
# 이전 라운드 기록 확인
ROUND=$(ls .brainstorm/rounds/ 2>/dev/null | wc -l)
ROUND=$((ROUND + 1))
echo "Round $ROUND"
```

라운드별 공격 강도:
- Round 1: 탐색 (가능성 중심)
- Round 2: 심문 (가정 해체)
- Round 3+: 생존 심사 (이게 진짜 될까?)

---

## Step 1 — EXPAND (병렬, 서브에이전트 4개)

4개 퍼소나가 동시에 현재 아이디어를 확장한다.

| 퍼소나 | 역할 | 출력 |
|--------|------|------|
| **연구자** | 유사 사례, 시장 데이터, 선례 | "이미 이렇게 한 사람들이 있다 / 없다" |
| **비판자** | 숨겨진 가정 공격, 반례, 최악의 시나리오 | "이게 틀릴 수 있는 이유 3가지" |
| **지지자** | 강점 극대화, 유추, 낙관 시나리오 | "이게 대박 날 수 있는 이유" |
| **조정자** | 절충안, 하이브리드, 피벗 방향 | "중간 지점 또는 다른 프레이밍" |

각 퍼소나: 3-5개 포인트, 핵심만, 군더더기 없이.

---

## Step 2 — ATTACK (병렬, 서브에이전트 3개)

생존 가능성을 데이터로 검증한다.

### 2A. Mom Test 시뮬레이션
Rob Fitzpatrick의 Mom Test 방법론 기반. 실제 고객 인터뷰를 시뮬레이션:
- "지금 이 문제를 어떻게 해결하고 있나요?" (현재 행동 질문)
- "마지막으로 이 문제로 골치 아팠던 게 언제였나요?" (과거 사실 질문)
- "이게 있다면 얼마를 낼 의향이 있나요?" (지불 의향 질문)
- "지금 당장 쓸 수 있다면 뭘 바꿔야 할까요?" (마찰 질문)

결과: 지불 의향 있음 / 없음 / 불명확 판정

### 2B. 9차원 승산 점수 (1-10점)
| 차원 | 질문 |
|------|------|
| 시장 크기 | 이걸 원하는 사람이 충분히 많은가? |
| 고통 강도 | 지금 당장 해결해야 할 만큼 아픈가? |
| 지불 의향 | 돈을 낼 만큼 중요한가? |
| 경쟁 강도 | 대안이 이미 충분히 좋은가? (낮을수록 좋음) |
| 실행 가능성 | 우리가 만들 수 있는가? |
| 타이밍 | 지금이 맞는 시점인가? |
| 차별화 | 남들이 쉽게 따라할 수 없는 무언가가 있는가? |
| 바이럴 | 자연스럽게 퍼질 수 있는가? |
| 창업자-시장 적합도 | 내가 이 문제를 직접 겪어봤는가? |

총점 = 합산 / 9. 7+ 진행 / 5-7 Pivot 검토 / 5 미만 중단 권고

### 2C. Reality Score (/intel 연동)
현재 `/intel` 데이터 또는 빠른 시장 스캔:
- 이미 존재하는 솔루션 수
- 최근 트렌드 방향 (상승/안정/하락)
- 시장 포화도

결과: 0-100점 + "이미 레드오션 / 블루오션 / 틈새 존재"

---

## Step 3 — SYNTHESIZE (Judge)

EXPAND + ATTACK 결과를 통합해 판정을 내린다.

출력 형식:
```
## Round N 브레인스토밍 결과

### 생존한 아이디어 (N개)
1. [아이디어] — 총점 X.X/10
   - 강점: ...
   - 근거: Mom Test O / Reality Score XX / 9차원 X.X

### Pivot 제안 (생존 못했지만 살릴 수 있는 것)
- [아이디어 A] → [피벗 방향]: ...

### 탈락 (이유)
- [아이디어 B]: [탈락 이유 한 줄]

### 다음 추천
- [ ] 프레토타입 유형 추천: [문짝 테스트 / 피노키오 / 가짜 랜딩 / YouTube MVP]
- [ ] 추가 라운드 필요 여부: Y/N
- [ ] 원래 단계로 복귀 시 집중할 것: ...
```

---

## Step 4 — 저장 + 복귀

```bash
mkdir -p .brainstorm/rounds
# .brainstorm/rounds/round-N-YYYY-MM-DD.md 에 결과 저장
```

복귀 선언:
```
Round N 완료. 원래 작업으로 돌아가세요.
계속하려면: /brainstorm [다음 아이디어 또는 피벗 방향]
```

---

## 프레토타입 유형 사전 (Judge가 추천 시 참조)

| 유형 | 설명 | 적합한 상황 |
|------|------|-----------|
| **문짝 테스트** | 존재하지 않는 제품의 문(랜딩)을 만들고 노크 수를 센다 | 수요 존재 여부 불명확 |
| **피노키오** | 가짜 제품을 진짜처럼 수동으로 운영한다 | 자동화 전 가치 검증 |
| **YouTube MVP** | 제품 없이 데모 영상만 올려 반응을 본다 | 비주얼이 핵심인 제품 |
| **가짜 랜딩** | 실제 기능 없이 가입/구매 버튼만 있는 페이지 | 전환율 측정 |
| **하루치 가짜 서비스** | 24시간 동안 사람이 직접 AI/자동화를 흉내낸다 | 자동화 ROI 판단 |
| **메카니컬 터크** | 내부 인력이 외부 사용자에게는 자동화처럼 보이게 운영 | 복잡한 서비스 핵심 검증 |

---

## 원칙

- 한 라운드 = 15분 이내 완료 목표
- 사용자가 선택하지 않아도 Judge가 최선을 추천
- 생존 아이디어 없어도 "왜 없는지"가 인사이트
- 라운드가 쌓일수록 공격이 세진다 — 살아남은 것은 진짜
- 코드 작성 금지 (이 모듈은 아이디어 검증만)
